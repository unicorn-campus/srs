> 후보 선정 과정에서 웹 검색으로 국내 실제 문제 상황 여부를 검증하고, 이미 상용 솔루션으로 해결된

## 후보1: MVNO 번호이동·개통 실패 실시간 감지 및 자동 재처리 API 구축

- **발주 배경(WHY)**: W3(실시간 프로비저닝 API)에 근거함. 번호이동 신청 시 필요한 ARS 사전동의 미완료,  
  기존 통신사 요금 미납, 번호이동 제한 부가서비스 가입 등으로 개통이 중간에 멈추는 사례가 빈번하나,  
  현재는 실패 사유를 사용자가 직접 ARS로 재확인하거나 CS에 문의해야 하는 수작업 구조라 개통 리드타임이  
  길어지고 실패 원인 파악이 지연됨(As-Is)
- **업무 요구**
  - 번호이동·개통 신청 접수 시 실패 사유(사전동의 미완료·요금 미납·번호이동 제한 등) 실시간 판별  
    (확인: 접수 후 5초 이내 실패 사유 코드 반환)
  - 실패 건 자동 재시도 및 고객 알림(SMS·앱 푸시) 발송 (확인: 실패 감지 후 1분 이내 알림 발송,  
    표본 100건 기준 재시도 성공률 측정·기록)
  - 개통 진행 상태(접수·사전동의·전산처리·완료) 실시간 대시보드 제공  
    (확인: 상태 변경 발생 후 3초 이내 대시보드 반영)
- **기술 요구**
  - MNO 번호이동·개통 시스템과의 실시간 연동 API (지표: 응답시간 2초 이내 / 피크 시간대 /  
    API 성공률 99.5% 이상)
  - 실패 사유 코드 표준화 및 Webhook 기반 이벤트 통지 (지표: 이벤트 유실률 0.1% 이하, 월 측정)
  - 이중화 구성을 통한 무중단 개통 처리 (지표: 장애 발생 시 자동 failover 30초 이내)
- **운용 요구**
  - 가용성 99.9% 이상 (연 다운타임 약 8시간 46분 이내)
  - 개통 실패 처리 SLA: 재시도 개시 1분 이내, 완전 실패 확정 시 고객 통지 5분 이내
  - 월간 개통 성공률·실패 유형별 통계 리포트 정기 제공
- **기대효과**: 번호이동·개통 실패 관련 CS 문의 40% 감소(현재 월 800건 → 480건),  
  평균 개통 처리 리드타임 2일 → 4시간 이내로 단축
- **시장 검증 근거**: 다수 알뜰폰 안내 매체에서 "번호이동 사전동의 실패"가 개통 지연의 대표 원인으로  
  반복 언급되며, 현재도 사용자가 ARS로 직접 재확인해야 하는 수동 프로세스로 남아 있어 실시간 자동  
  감지·알림 솔루션은 상용화되어 있지 않음을 확인함

## 후보2: MVNO 도매 회선 품질(QoS)·용량 실시간 가시성 대시보드 구축

- **발주 배경(WHY)**: W5(도매 용량·트래픽 가시성)에 근거함. 2022년 9월 도매제공 의무제가 일몰된 데 이어  
  2026년 4월부터는 MVNO가 MNO와 도매대가·용량을 직접 협상해야 하는 사후규제 구조로 전환되고  
  협상 기한도 90일에서 60일로 단축됨. 그러나 MVNO는 자사 가입자가 실제 사용 중인 도매 대역의  
  실시간 품질·용량 데이터를 MNO로부터 제공받지 못해 협상·품질 대응이 사후적으로만 이루어짐(As-Is)
- **업무 요구**
  - 지역·시간대별 도매 대역 트래픽 사용률 조회 (확인: 조회 요청 후 3초 이내 응답)
  - 가입자 체감 품질 지표(지연·손실률·처리속도) 조회 및 임계치 초과 시 알림  
    (확인: 임계치 초과 감지 후 5분 이내 알림 발송)
  - 용량 협의용 리포트 자동 생성(피크타임 혼잡 지역 상위 10곳) (확인: 일 1회 자동 생성,  
    수기 집계 대비 오차 0%)
- **기술 요구**
  - MNO 네트워크 장비(NMS·TMS)로부터 표준 API(NETCONF·REST) 기반 트래픽 데이터 연동  
    (지표: 데이터 수집 주기 5분 이내, 연동 성공률 99.5% 이상)
  - 대용량 시계열 데이터 저장·조회 구조 (지표: 최근 3개월치 데이터 조회 시 응답 3초 이내)
  - MVNO별 데이터 접근권한 분리·격리 및 전송구간 암호화(TLS 1.2 이상)  
    (지표: 타 MVNO 데이터 접근 차단율 100%)
- **운용 요구**
  - 가용성 99.9% 이상 (연 다운타임 약 8시간 46분 이내)
  - 데이터 갱신 지연 통지 15분 이내, MTTR 4시간 이내
  - 분기별 협상용 품질·용량 리포트 정기 제공 및 이력 최소 1년 보관
- **기대효과**: 용량 협의 소요기간 50% 단축(현재 평균 40일 → 20일), 품질 관련 고객 불만  
  이관 건수 30% 감소(월 20건 → 14건)
- **시장 검증 근거**: 2025년 2월 도매제공 관련 고시 개정으로 협상 기한이 90일에서 60일로 단축되고  
  2026년 4월부터 MVNO 직접 협상 사후규제로 전환됨이 확인됨. 국내 트래픽 모니터링 솔루션(하몬소프트  
  Netis-TMS, 넷맨 SOLOMON 등)은 존재하나 모두 통신사(MNO) 내부 운영용이며, MVNO에 도매 회선  
  품질·용량을 실시간으로 노출하는 상용 솔루션은 확인되지 않아 유효한 미해결 과제로 판단함
