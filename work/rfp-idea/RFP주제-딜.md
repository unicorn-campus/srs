# RFP 주제 — 딜 (비용·SLA·계약 관점)

## 후보1: 도매대가 협상 지원 — 코스트플러스 원가검증 시뮬레이션 시스템

- **발주 배경(WHY)**: W1(망 도매 정산 자동화) 확장. 2025년 4월부터 알뜰폰 도매대가 산정 방식이 정부 개입형  
  사전규제에서 MVNO-MNO 개별 협상 기반 사후규제로 전환됨. 정부는 협상 기한을 60일 이내로 제한하고 망 원가에  
  이윤을 더하는 '코스트플러스' 방식 적용을 허용했으나, MNO가 망 원가(감가상각비·무형자산상각액 등)를 선제  
  공개해야만 검증이 가능한 구조라 산정기준 전환 자체가 어려운 상황임. MVNO는 자체적으로 원가 요소별 도매대가  
  적정성을 검증할 수단이 없어 협상 때마다 MNO 제시안을 그대로 수용하거나, 근거 부족으로 60일 기한 내 협상을  
  마무리하지 못하는 리스크에 노출됨(As-Is)
- **업무 요구**
  - 요금제·가입자 세그먼트별 도매대가 방식(종량제 RM / 수익배분형 정액제 RS) 손익 시뮬레이션  
    (확인: 표본 요금제 20종 시뮬레이션 결과와 실 정산액 오차 5% 이내)
  - 코스트플러스 원가 구성요소(설비상각비·운영비·이윤율 등)별 입력값 대비 도매대가 적정성 검토 리포트 생성  
    (확인: 원가 항목 10종 이상 반영, 리포트 생성 요청 후 10분 이내 산출)
  - 협상 진행 단계·기한(제안·검토·합의) 관리 및 60일 법정 기한 잔여일수 자동 경고  
    (확인: 기한 도래 D-10/D-3 시점 자동 알림 발송)
- **기술 요구**
  - 원가·정산 데이터 입력 및 시뮬레이션 결과 조회 웹 기반 시스템 (지표: 요금제 100종 동시 시뮬레이션 처리  
    10분 이내 / 협상 준비 피크 기간 기준)
  - 외부(공시자료·통계청 등) 원가 참조 데이터 연계 API (지표: 데이터 갱신 주기 월 1회 이상, 연계 성공률  
    99% 이상)
  - 협상 근거자료·시뮬레이션 이력 데이터 암호화 저장 및 접근권한 3단계 분리(관리자·협상담당자·조회자)  
    (지표: 전송구간 TLS 1.2 이상, 저장 데이터 암호화)
  - 시스템 장애 시 협상 이력·시뮬레이션 데이터 자동 백업 (지표: 일 1회 백업, 복구시간 4시간 이내)
- **운용 요구**
  - 시스템 가용률 99.9% 이상 (연 다운타임 약 8시간 46분 이내)
  - 원가 참조 데이터·시뮬레이션 로직 변경 시 사전 통지(최소 7일 전) 및 변경 이력 관리
  - 협상 담당자 대상 초기 도입 교육 1회 이상 실시 및 사용자 매뉴얼 제공
  - 협상 이력·산정 근거 자료 5년 이상 보관(분쟁 발생 시 사후규제 대응 근거 확보)
- **기대효과**: 협상 준비 기간 평균 30일 → 10일로 단축, 60일 법정 기한 내 협상 완료율 60% → 95%로 개선
- **시장 검증 근거**: 전자신문 "알뜰폰 도매제공 60일 이내로, 망 원가 반영해 요금 낮추는 '코스트 플러스' 방식  
  적용"(2025.02.17, https://www.etnews.com/20250217000254), 디지털투데이 "알뜰폰 도매대가 '코스트 플러스'  
  탄력…정부, 고시·시행령 개정"(https://www.digitaltoday.co.kr/news/articleView.html?idxno=553520), 디지털투데이  
  "과기정통부 알뜰폰 도매대가 인하, 코스트 플러스 방식 도입할까"  
  (https://www.digitaltoday.co.kr/news/articleView.html?idxno=525986)에서 "이통사들이 먼저 망 원가를 공개해야  
  가능한 부분이기에 산정기준 전환이 쉽지 않을 전망"이라고 보도해 원가 투명성·검증 공백이 실제 쟁점임을 확인함.  
  디지털데일리 "[알뜰폰 위기 진단]② 마지막 도매대가 협상 시작…산정기준 바뀌어야"  
  (https://m.ddaily.co.kr/page/view/2024052908564910671)도 산정기준 개편 필요성을 보도함. 검색 결과 MVNO향  
  원가검증·협상지원 전용 상용 솔루션은 확인되지 않아 자체 구축 수요가 유효함

## 후보2: 도매·정산 핸드오프 자동 대사 및 청구오류 검증 시스템

- **발주 배경(WHY)**: W1(망 도매 정산 자동화) 확장. 번호이동 시 이전 통신사(MNO 또는 타 MVNO) 사용 요금이  
  '번호이동 전 사용 요금'으로 새 MVNO 청구서에 합산 승계되는 정산 핸드오프 구조상, 사용량 원장 오류나  
  중복·과다 산정이 발생해도 MVNO가 이를 자동으로 검출하지 못하고 가입자 민원이 접수된 후에야 사후 확인하는  
  구조임. 정산 대사가 수작업 위주라 오류 확인·정정에 장기간이 소요됨(As-Is)
- **업무 요구**
  - 번호이동·해지 시점의 일할 정산 원장과 신규 MVNO 청구 데이터 자동 대사 (확인: 대사 불일치 건 자동  
    검출율 99% 이상)
  - 이관 사업자별(MNO/타 MVNO) 정산 데이터 형식 표준화 및 매핑 관리 (확인: 사업자 5개사 이상 데이터  
    포맷 정상 매핑)
  - 청구오류 의심 건 자동 플래깅 및 담당자 검토 큐 제공 (확인: 플래깅 후 검토 착수까지 평균 1영업일 이내)
- **기술 요구**
  - 번호이동관리기관(MNP) 및 MNO 정산 시스템 연계 API (지표: 연계 성공률 99.5% 이상 / 월 평균 / 배치  
    실패 시 자동 재시도 3회)
  - 대사 배치 처리 성능 (지표: 일 정산 건수 10만 건 기준 배치 완료 4시간 이내 / 월말 정산 피크 기준)
  - 정산·개인정보(가입자 식별정보 포함) 데이터 암호화 및 접근권한 분리(관리자·정산담당자·조회자 3단계)  
    (지표: 전송구간 TLS 1.2 이상, 개인정보 마스킹 처리)
  - 대사 배치 실패 시 자동 재처리 및 실패 이력 알림 (지표: 실패 감지 후 30분 이내 담당자 알림)
- **운용 요구**
  - 정산 대사 시스템 가용률 99.9% 이상 (연 다운타임 약 8시간 46분 이내)
  - 청구오류 의심 건 통지 1영업일 이내, 정정 처리 MTTR 3영업일 이내
  - 가입자 이의신청 접수 시 조사 결과 10일 이내 회신(전기통신사업법 관행 기준 준수)
  - 정산·대사 이력 및 처리 로그 3년 이상 보관, 반기 1회 대사 정확도 자체 점검 실시
- **기대효과**: 청구오류 발견까지 소요기간 평균 15일 → 2일로 단축, 청구오류 관련 가입자 민원 월 50건 →  
  10건으로 감소
- **시장 검증 근거**: 클리앙 이용자 게시글 "알뜰폰 번호이동 후 요금 과다 청구(진행중)"  
  (https://www.clien.net/service/board/use/12847911)에서 헬로모바일→안심모바일 번호이동 과정 중 요금제  
  기준 과다 청구 사례가 실제로 보고됨. 꼬미짱 블로그 "통신사 옮기면 요금이 이중청구 될까? 알뜰폰 번호이동  
  전 꼭 알아야 할 팩트 체크"에서도 번호이동 전 사용 요금이 신규 통신사 청구서에 '번호이동 전 사용 요금'으로  
  합산 승계되는 구조 자체가 청구 오류·민원의 소지가 됨을 설명함. 머니투데이 "'영세업체 고사' vs '부실기업  
  빨대', 알뜰폰 도매대가 뭐길래"(https://www.mt.co.kr/tech/2025/01/09/2025010714540998225)에서도 정산  
  대사가 수작업 위주임을 시사함. 해당 핸드오프 구간 자동 대사·오류검증을 표준 제공하는 상용 솔루션은  
  검색상 확인되지 않아 자체 구축 수요가 유효함
