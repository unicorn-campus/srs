# RFP 주제 — 리서처 (규제·요구사항 관점)

> 발상 관점: 개인정보보호법·전기통신사업법·도매제공 의무 등 규제 준수형 요구 중심  
> 검증 방법: 각 후보를 웹 검색으로 국내 실제 발생 여부·미해결 여부 확인, 근거는 각 후보 하단에 기재

## 후보1: MVNO 비대면 개통 채널 안면인증·실명확인 연동 시스템 구축

- **발주 배경(WHY)**: W3(실시간 프로비저닝 API) 확장 + 규제 준수. 2024년 대포폰 적발 건수가 97,399건으로  
  2020년(8,923건) 대비 약 11배 급증했고, 이 중 알뜰폰을 통한 부정개통이 89,927건(92.3%)을 차지함.  
  정부는 2025년 12월 안면인증을 시범 도입하고 2026년 3월 정식 의무화를 예고했으나, 안면인증·본인확인  
  엔진은 MNO 자체 채널 위주로 구축되어 MVNO 비대면 개통 프로세스에는 실시간 연동이 되지 않아 심사  
  지연·명의도용 개통이 반복됨(As-Is)
- **업무 요구**
  - 비대면 개통 신청 시 신분증 진위확인과 안면인증(라이브니스 검증) 실시간 처리  
    (확인 기준: 요청 후 5초 이내 인증결과 응답, 위변조 신분증 검출율 99% 이상)
  - 명의도용방지서비스(M-Safer) 연계로 동일 명의 다회선 개통 시도 조회·경고  
    (확인 기준: 180일 내 4회선 초과 개통 시도 자동 차단율 100%)
  - 개통 거절·보류 이력과 사유를 MVNO 운영 포털에 실시간 통지  
    (확인 기준: 거절 발생 후 1분 이내 알림 도달)
  - 안면인증 실패 시 상담원 영상통화 등 대체 인증 수단 자동 전환 플로우 제공  
    (확인 기준: 실패 감지 후 대체 인증 수단 전환까지 10초 이내)
- **기술 요구**
  - MNO 안면인증 엔진과 MVNO 개통 시스템 간 API 연동 (지표: 초당 200건 처리 / 피크 시간대 / 응답 5초 이내)
  - 안면데이터·신분증 사본 전송구간 TLS 1.3 암호화, 저장 시 AES-256 암호화, 접근 이력 로그 5년 보존
  - ISMS 인증 및 CISO 지정 요건을 충족하는 시스템 통제항목 반영 (지표: ISMS 인증 심사 통과 여부)
- **운용 요구**
  - 인증 시스템 가용성 99.95% 이상 (연 다운타임 약 4시간 22분 이내)
  - 안면인증 오류·장애 통지 15분 이내, MTTR 2시간 이내
  - 전기통신사업법 시행령 등 법령 개정 시 정책 파라미터 무중단 반영, 월 1회 이상 규제준수 점검 리포트 제공
- **기대효과**: 알뜰폰 부정개통 비중 92.3% → 50% 이하로 축소, 개통 심사 처리시간 평균 1일 → 10분 이내 단축

  - [휴대전화 개통 시 '안면인증'…"본인여부 한번 더 확인" - 정책브리핑](https://www.korea.kr/news/policyNewsView.do?newsId=148956798)

  - 결론: 상용 통합 솔루션이 아직 없고 정책·법령이 진행 중인 미해결 문제로, RFP 주제로 적합함

## 후보2: 도매제공의무 사후규제 대응 도매대가 산정·협상 관리 시스템 구축

- **발주 배경(WHY)**: W1(망 도매 정산 자동화) 확장 + 규제 준수(전기통신사업법 제38조의2). 2025년 시행령  
  개정으로 도매대가 협상이 사전규제에서 사후규제로 전환되며 협상 기한이 90일에서 60일 이내로 단축되고,  
  기존 리테일마이너스 방식에 코스트플러스 방식이 추가되어 산정 근거자료 요구가 복잡해짐. 현재 MNO-MVNO  
  간 산정 근거자료 공유가 수기·비정형 문서로 이뤄져 협상 지연과 정산 분쟁이 반복됨(As-Is)
- **업무 요구**
  - 리테일마이너스·코스트플러스 방식별 산정 근거자료(원가·회피가능비용 내역) 자동 산출·제공  
    (확인 기준: 자료 요청 후 3영업일 이내 제공)
  - 협상 진행 단계(요청-자료제공-협의-체결) 마일스톤 관리 및 60일 기한 임박 자동 알림  
    (확인 기준: 기한 도래 D-10/D-3 자동 알림 발송율 100%)
  - 금지행위(부당 정보요구, 산정주기 일방 변경 등) 의심 이력 자동 탐지·월간 리포트 생성  
    (확인 기준: 협정 위반 의심 건 월 1회 이상 자동 리포트 산출)
  - 도매대가 변경 이력·협정 문서 버전관리 및 MVNO 열람 포털 제공 (확인 기준: 열람 요청 후 1일 이내 응답)
- **기술 요구**
  - 정산·원가 데이터 배치+실시간 하이브리드 연동 (지표: 일 배치 정산 데이터 매일 06시까지 100% 적재)
  - 산정 근거자료 위변조 방지 전자서명·해시 검증, 접근권한 역할기반통제(RBAC) 3단계 적용
  - 규제기관 자료제출 요청 대응 API 제공, 감사로그(Audit Log) 3년 이상 변경불가 저장구조로 보존
- **운용 요구**
  - 정산 시스템 가용성 99.9% 이상 (연 다운타임 약 8시간 46분 이내)
  - 정산 데이터 대사 불일치 발견 시 4시간 이내 1차 소명자료 제공(MTTR)
  - 협상 기한 미준수·분쟁 발생 시 에스컬레이션 절차 가동, 법무 검토 요청 접수 후 1영업일 이내 착수
- **기대효과**: 도매대가 협상 평균 소요기간 90일 → 45일 이내 단축, 월간 정산 분쟁 건수 15건 → 3건 이하로 축소

## 후보 간 차별성

- 후보1은 **가입자 개통 시점의 본인확인·부정방지**(BSS 개통 프로세스) 영역  
- 후보2는 **MNO-MVNO 간 도매대가 산정·협상·정산**(B2B 계약·정산) 영역  
- 두 주제 모두 W1~W5 실수요에 근거하되 서로 다른 업무 도메인(개통 vs 정산)을 다루어 중복 없음
