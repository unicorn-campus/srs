# RFP 주제 — 아키 (기술 아키텍처 관점)

## 후보1: MNO 전산 이중화 기반 MVNO 실시간 프로비저닝 API 게이트웨이 구축

- **발주 배경(WHY)**: W3(실시간 프로비저닝 API) 근거. MVNO의 유심 개통·번호이동·부가서비스 신청이 MNO 단일  
  전산망 경로에만 의존해, MNO측 정기 시스템개편·장애 발생 시 MVNO의 개통·조회 업무가 전면 중단됨(As-Is).  
  대체 경로(비동기 접수·이중화 게이트웨이)가 없어 해당 시간 동안 신규 가입·번호이동 접수 자체가 불가능함
- **업무 요구**
  - 유심 개통·번호이동·부가서비스 신청을 API로 실시간 처리 (확인: 요청 후 5초 이내 처리결과 회신, 성공률 99% 이상)
  - MNO 정기점검·시스템 개편 공지 시에도 MVNO 접수창구가 비동기 큐잉으로 계속 동작 (확인: 계획정지 시간 중 접수 유실 0건)
  - 개통 실패 건 표준 실패사유 코드 제공 및 자동 재시도 (확인: 실패사유 코드 20종 이상, 재시도 후 최종 실패율 1% 이하)
- **기술 요구**
  - API 게이트웨이 Active-Active 이중화 구성 (지표: 장애조치 자동전환 30초 이내 / 이중 데이터센터·AZ 구성)
  - 개통 API 처리성능 300 TPS 이상 (측정조건: 피크 시간대, 응답시간 5초 이내)
  - 계획정지(전산개편) 사전 통보 API 및 최소 D-7 공지 체계, 정지 윈도우 중 비동기 접수 큐 자동 재처리
  - mTLS + OAuth 2.0 인증, IP 화이트리스트, 접근권한 3단계(관리자·운영자·조회자) 적용
- **운용 요구**
  - 가용성 99.95% 이상 (연간 다운타임 약 4시간 23분 이내)
  - 계획정지 시에도 대체경로(비동기 접수) 운영, 무중단 전환 리허설 연 2회 실시 (확인: 리허설 성공률 100%)
  - 장애 통지 15분 이내, MTTR(평균복구시간) 2시간 이내
  - 월간 이중화 전환 테스트 결과 및 API 가용률·처리성공률 리포트 제공
- **기대효과**: 전산개편·장애로 인한 MVNO 업무 중단시간 연 72시간 → 4시간 이내로 축소, 개통 실패 재접수  
  처리시간 평균 2일 → 4시간 이내로 단축
- **시장 검증 근거**: LG U+가 2023년 10월 20일 22시~23일 08시 전산 시스템 개편을 진행하며 홈페이지·앱·개통·  
  요금납부 등 전체 서비스가 일시 중단됨  
  ("유플러스 시스템 개편 중 서비스 일시 중단 안내", LG U+ 공식 공지, https://www.lguplus.com/support/service/notice/2000018994).  
  2012년 3월에도 유사한 전산개편발 개통·요금납부 중단 사례가 있었음  
  ("LG U+/논란 및 사건 사고", 나무위키, https://namu.wiki/w/LG%20U+/%EB%85%BC%EB%9E%80%20%EB%B0%8F%20%EC%82%AC%EA%B1%B4%20%EC%82%AC%EA%B3%A0).  
  MVNE(스테이지파이브·Circles 등)는 Full MVNO향 코어망·멀티테넌시 플랫폼을 제공하나, 기존 MNO 도매망에  
  의존하는 일반 MVNO의 MNO측 전산개편·장애에 대한 이중화·장애격리 문제는 별도로 해결되어 있지 않아  
  상용 솔루션으로 충분히 해결된 문제가 아님을 확인함

## 후보2: MVNO향 실시간 도매 용량·QoS 가시성 API/대시보드 구축

- **발주 배경(WHY)**: W5(도매 용량·트래픽 가시성) 근거. 국내 알뜰폰은 도매 대역(특정 용량)만 임대해 사용하는  
  구조라 트래픽이 집중되면 품질이 저하될 수 있으나, MVNO는 도매 대역의 실시간 사용률·QoS·혼잡도를 확인할  
  방법이 없어 품질저하를 가입자 민원 접수 후에야 사후적으로 인지함(As-Is). 도매대가도 2022년 협상가격에  
  정체되어 있어 정량 데이터 기반 협상 근거자료 확보가 더욱 중요해졌음
- **업무 요구**
  - 도매 대역별 실시간 사용률·혼잡도 조회 (확인: 요청 후 3초 이내 응답, 5분 단위 갱신)
  - 시간대별·지역별 QoS 지표(지연·손실률) 추이 대시보드 제공 (확인: 최근 30일 이력 조회, 이상치 자동 하이라이트)
  - 용량 임계치 초과 시 사전 경보 알림 (확인: 임계치(예: 사용률 80%) 도달 시 5분 이내 알림 발송)
- **기술 요구**
  - MNO 코어망 트래픽 수집 시스템과 연동하는 실시간 데이터 파이프라인 구축 (지표: 수집 지연 1분 이내, 정확도 99.5% 이상)
  - 대시보드·API 조회 성능 200 TPS 이상 (측정조건: 피크 시간대, 응답 3초 이내)
  - 클라우드 기반 시계열 데이터 저장소 구성, 원본 데이터 최소 13개월 보관(정량 협상 근거 자료용)
  - API 인증 OAuth 2.0 및 TLS 1.2 이상 암호화, 사업자별 데이터 접근 격리(멀티테넌시)
- **운용 요구**
  - 대시보드·API 가용성 99.9% 이상 (연간 다운타임 약 8시간 46분 이내)
  - 데이터 지연·누락 발생 시 통지 30분 이내, MTTR 4시간 이내
  - 분기별 도매 용량 협의용 정량 리포트(사용률·QoS 추이) 정기 제공
  - 신규 MVNO 온보딩 시 대시보드·API 계정 발급 및 교육을 5영업일 이내 완료
- **기대효과**: 품질저하 관련 고객 민원 원인파악 소요시간 평균 5일 → 3시간 이내로 단축, 도매 용량  
  협상 시 정량 데이터 첨부율 0% → 100%로 개선되어 협상 준비기간 평균 20일 → 10일로 단축
- **시장 검증 근거**: 알뜰폰은 통신3사와 동일 기지국 망을 임대하나 특정 대역만 할당받는 구조라 트래픽 집중 시  
  품질저하가 발생할 수 있는 구조적 문제가 존재함  
  ("알뜰폰 통화품질과 데이터 속도에 대한 오해와 진실", 모요, https://www.moyoplan.com/tips/알뜰폰-이용-방법/fact-check).  
  2024년 3월 기준 도매단가가 2022년 협상가격에 정체되어 있고 도매제공 의무제 일몰(2022.9)로 정부 대리협상  
  근거가 사라져 사업자간 자율협상 부담이 커짐  
  ("'싸게 쓰자' 고객 몰려갔는데…'망할 위기' 알뜰폰 '곡소리' 나는 이유", 머니투데이, https://news.mt.co.kr/mtview.php?no=2025032916280063914).  
  KAIT '알뜰폰 허브'는 요금제 비교·가입신청 API만 제공하며 실시간 용량·QoS 가시성 기능은 제공하지 않아  
  상용 솔루션으로 충분히 해결된 문제가 아님을 확인함
