# RFP 주제 — 케어 (운영·사후관리 관점)

> 관점: 장애 대응·모니터링·고객 운영 지속성 중심. 계약 이후 실제 운영 국면에서 반복적으로  
> 발생하는 MVNO의 사후관리 공백을 근거로 주제를 발상함.

---

## 후보1: MVNO향 통신 장애·보안사고 실시간 통지 및 영향범위 조회 시스템

- **발주 배경(WHY)**: W5(도매 용량·트래픽 가시성)를 장애·사고 통지 영역으로 확장한 실수요임.  
  현재 MNO의 대규모 망 장애·보안사고 발생 시 MVNO는 공식 통지를 실시간으로 받지 못하고  
  뉴스·SNS·자체 모니터링으로 뒤늦게 인지하는 구조임(As-Is). 이 때문에 2025년 4월 SKT 유심 정보  
  유출 사고 당시 KB리브엠 등 SKT망 알뜰폰 사업자 고객센터가 문의 폭주로 마비됐고, 2018년 KT  
  아현지사 화재 때도 알뜰폰 가입자 대상 보상 안내가 사고 발생 약 3주 후에야 개별 공지됨. 통지  
  지연이 MVNO 고객센터 마비와 가입자 이탈(SKT 사고 직후 3일간 9만명 이상 이탈)로 직결됨
- **업무 요구**
  - 장애·보안사고 발생 즉시 심각도별 자동 통지 (확인: 사고 인지 후 5분 이내 MVNO 담당자 통지)
  - 영향 가입자 규모·지역·서비스 범위 실시간 조회 (확인: 요청 후 3초 이내 응답, 표본 100건 100% 일치)
  - 복구 진행률·예상 복구시각 갱신 제공 (확인: 15분 주기 갱신, 복구완료까지 이력 연속 조회 가능)
  - 사고 종료 후 장애 이력·재발방지대책 리포트 자동 생성 (확인: 종료 후 3영업일 이내 제공)
- **기술 요구**
  - Webhook/API 기반 실시간 이벤트 푸시 (지표: 발생~통지 지연 1분 이내 / 성공률 99.9% / 월 측정)
  - 대규모 동시 조회 처리 300 TPS 이상 (측정조건: 장애 피크시간대, 응답 2초 이내)
  - 다중 채널 통지(API+이메일+SMS) 이중화, 단일 채널 장애 시 자동 전환 (지표: 전환 30초 이내)
  - 접근 인증 OAuth 2.0, 전송구간 암호화(TLS 1.2 이상), 통지 이력 위변조 방지 로그 보관
- **운용 요구**
  - 통지 시스템 자체 가용성 99.95% 이상 (연 다운타임 약 4시간 22분 이내)
  - 장애 통지 SLA 준수율 월 98% 이상, 미준수 시 페널티 조항 반영
  - 24x365 공동 대응 핫라인 운영 및 분기 1회 합동 장애대응 훈련(모의훈련) 실시
  - 월간 장애대응 리포트(통지 소요시간·MTTR·재발률) 제공
- **기대효과**: 장애 인지~통지 소요시간 평균 40분 → 5분 이내 단축, 장애 발생 시 MVNO 고객센터  
  문의량 30% 감소(예: 월 5,000건 → 3,500건)

  - 결론: 실제 문제 확인됨. 대규모 사고 시 MVNO향 공식 실시간 통지 체계는 국내에 상용화되어  
    있지 않으며, 사고 때마다 반복적으로 통지 지연·고객센터 마비가 재현됨

## 후보2: MVNO 고객센터용 가입자별 서비스 장애 실시간 진단 포털

- **발주 배경(WHY)**: W3(실시간 프로비저닝)·W4(실시간 사용량 조회)를 사후 진단 영역으로 확장한  
  실수요임. MVNO는 자체 망·NOC이 없어 유심 불량·개통 지연·품질저하 등 개별 가입자 이슈가  
  접수돼도 MNO 측에 문의해야만 원인을 확인할 수 있음(As-Is). 소비자단체 조사(소비자가 만드는  
  신문, 2024년 결산-통신)에 따르면 알뜰폰 시장 확대 과정에서 AS 인프라 미비로 "고객센터 불통,  
  개통 지연, 품질 불량"이 최다 민원 유형으로 나타났으며, 유심 불량·개통 지연으로 서비스를 이용  
  하지 못했다는 불만이 가장 흔한 것으로 확인됨. 원인 진단 도구 부재가 상담 장기화·재문의를 유발함
- **업무 요구**
  - 가입자 번호 기준 회선상태(활성/정지/장애)·유심 상태 실시간 조회 (확인: 3초 이내 응답)
  - 최근 24시간 내 개통·인증·과금 오류 이력 조회 (확인: 표본 100건 대사 결과 100% 일치)
  - 기지국·권역 단위 품질저하(호절단률·데이터속도 저하) 여부 조회 및 알림 (확인: 임계치 초과 시  
    5분 이내 CS 담당자 알림)
  - 진단 결과 기반 1차 조치 가이드(표준 응대문구) 자동 제공
- **기술 요구**
  - MNO OSS/NOC 연동 조회 API 응답시간 2초 이내 (지표: 성공률 99.9% / 정상부하 / 월 측정)
  - 동시 조회 처리 500 TPS 이상 (측정조건: 전체 CS 상담원 동시 접속 피크시간대)
  - 상담원 권한별 3단계 접근제어(관리자·상담원·조회자), 조회 이력 감사로그 1년 보관
  - 야간·공휴일 포함 24x365 실시간 API 제공(배치 처리 미의존)
- **운용 요구**
  - 포털 가용성 99.9% 이상 (연 다운타임 약 8시간 46분 이내)
  - 장애 통지 30분 이내, MTTR 4시간 이내
  - 신규·기존 CS 상담원 대상 분기 1회 교육 및 진단 매뉴얼 갱신
  - 월간 이용 통계(조회건수·1차 해결률) 리포트 제공, 1차 해결률 목표 70% 이상
- **기대효과**: CS 1건당 평균 처리시간 12분 → 6분 단축, 동일 건 재문의(콜백) 비율 25% → 10%로 감소

    소비자가만드는신문, 2024](https://www.consumernews.co.kr/news/articleView.html?idxno=721599)
  - [한국소비자원, "알뜰폰서비스 이용실태 및 개선방안"](https://www.kca.go.kr/home/board/download.do?menukey=4062&fno=10008478&bid=00000146&did=1001436795)
  - 결론: 실제 문제 확인됨. 개별 가입자 단위 장애·품질 진단을 MVNO CS가 자체적으로 수행할 수  
    있는 상용 포털은 확인되지 않았고, 소비자단체 조사에서도 개통 지연·유심 불량·품질 불량이  
    구조적 최다 민원으로 반복 확인됨(후보1은 대규모·망 전체 장애의 사후 통지, 후보2는 개별  
    가입자 단위 상시 진단으로 문제 영역이 서로 다름)
