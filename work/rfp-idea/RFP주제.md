> 작성: 오케스트레이터(클로니) — 6명(리서처·아키·스토리텔러·데모·딜·케어) 투표 결과 집계  
> 대상: `work/rfp-idea/RFP주제후보.md`의 통합 후보 9건(C1~C9)  
> 목적: 통신사 B2B 기획자 대상 Claude Code 활용 SRS 작성 교육에 사용할 교육용 가상 RFP의  
> 기반 주제 1건을 최종 선정함

## 1. 투표 규칙

- 투표권: 1인당 비즈니스 가치 3표 + 실현 가능성 3표(두 축을 각각 행사, 1인 총 6표)
- 이해상충 배제: 자신이 제안자(병합 포함)로 표기된 후보에는 투표 불가
  - 리서처 제외 대상: C1, C2 / 아키 제외 대상: C3, C4 / 스토리텔러 제외 대상: C5, C6
  - 데모 제외 대상: C1, C3 / 딜 제외 대상: C2, C7 / 케어 제외 대상: C8, C9
- 분산 규칙: 각 축에서 한 후보에 최대 2표까지 분산 가능
- 집계: 후보별로 두 축 득표를 각각 합산한 뒤 (비즈니스 가치 + 실현 가능성) 총점 산출
- 선정: 총점 최고 주제 1건. 동점 시 실현 가능성 득표 우선, 그래도 동점이면 리서처가 사유를 명시해 결정

## 2. 팀원별 투표 상세

### 리서처 (제외: C1, C2)

- 비즈니스가치: C3 1표, C5 1표, C8 1표
- 실현가능성: C8 2표, C5 1표
- 사유: 비즈니스가치는 C3(2026년 4월 사후규제 전환·60일 기한이라는 명확한 규제 시급성), C5(SKT가  
  경쟁사 인식으로 통합조회 플랫폼을 의도적으로 미구축했다는 구체적 실수요 근거), C8(2025년 SKT  
  유심유출·2018년 KT 아현화재라는 실제 사고 사례 기반의 강한 교육 설득력)에 고르게 배분함.  
  실현가능성은 C8에 2표를 집중했는데, Webhook/API 기반 이벤트 통지라는 단일 연동점 구조가 SRS  
  실습 범위로 가장 깔끔하고 성공지표(지연시간·성공률)가 명확하기 때문임. C5에도 1표를 배정했는데  
  PCRF/PCF 단일 연동과 임계치 알림 로직이 명세가 명확하고 실습 난이도가 적절하기 때문임

### 아키 (제외: C3, C4)

- 비즈니스가치: C2 2표, C1 1표
- 실현가능성: C9 2표, C5 1표
- 사유: 비즈니스가치는 2025년 시행령 개정으로 협상기한이 60일로 단축된 C2가 규제 준수 시급성과  
  코스트플러스 방식 도입이라는 명확한 정책 실수요를 갖춰 최우선으로 판단했고, 대포폰 92.3%라는  
  구체적 수치와 전기통신사업법 근거를 갖춘 C1을 다음으로 평가함. 실현가능성은 C9(가입자 상태·이력  
  단순 조회 API, OSS 연동 명세 명확, 3단계 권한 구조)가 SRS 실습 범위로 복잡도가 적절해 최우선으로  
  선정했으며, PCRF/PCF 연동이 비교적 표준화되어 있고 임계치 알림 로직이 명확한 C5를 차순위로 평가함

### 스토리텔러 (제외: C5, C6)

- 비즈니스가치: C2 2표, C8 1표
- 실현가능성: C4 1표, C9 1표, C3 1표
- 사유: 비즈니스가치는 C2에 집중함. 협상기한 법정 단축과 코스트플러스 방식 신설이라는 명백한 규제  
  준수 시급성과, MNO 원가 비공개 구조에서 MVNO 자체 검증수단이 없다는 실수요 근거가 뚜렷해 교육  
  사례로 설득력이 높음. C8은 SKT 유심 유출·KT 아현화재라는 실제 사건 두 건을 근거로 제시해 "왜  
  지금 필요한가"를 가장 생생하게 전달할 수 있는 소재라 1표를 배정함. 실현가능성은 C4·C9·C3에  
  분산함. 세 후보 모두 API 게이트웨이 이중화, 조회 포털, 대시보드라는 비교적 단순한 도메인이면서  
  TPS·응답시간·failover 시간 등 정량 지표가 이미 명확히 제시돼 있어 실습 범위로 적절함

### 데모 (제외: C1, C3)

- 비즈니스가치: C8 2표, C5 1표
- 실현가능성: C4 2표, C9 1표
- 사유: C8은 2025년 SKT 유심유출 사고·2018년 KT 아현화재 등 실제 발생 사건을 근거로 실수요와  
  시급성이 매우 구체적이고 교육 스토리텔링 소재로도 설득력이 커서 2표를 배정함. C5는 SKT가 알뜰폰  
  통합 사용량 조회를 아예 제공하지 않는다는 구조적 공백이 명확해 1표를 부여함. 실현가능성 측면에서  
  C4는 API 게이트웨이·이중화·TPS·failover 등 지표가 이미 정량적으로 명확하고 기술 난이도가 교육용  
  SRS 실습 범위에 적합해 2표를 줌. C9는 MNO OSS/NOC 조회 위주의 단순 읽기 API 구조로 복잡도가  
  낮아 실습용으로 적절하여 1표를 부여함

### 딜 (제외: C2, C7)

- 비즈니스가치: C1 2표, C8 1표
- 실현가능성: C5 1표, C8 1표, C9 1표
- 사유: C1은 2024년 대포폰 92.3%가 알뜰폰에 집중된다는 구체 수치로 규제 준수 시급성과 실수요  
  근거가 가장 명확해 2표를 집중함. C8은 SKT 유심 유출·KT 아현 화재라는 실제 사고 사례를 근거로  
  들어 실수요 설득력이 강해 1표를 배정함. 실현가능성 축에서는 C1처럼 연동 대상이 많은 후보는  
  교육용 SRS 실습 범위를 넘어서는 리스크가 있어 배제함. 대신 C5(PCRF/PCF 단일 연동·임계치 알림),  
  C9(OSS/NOC 조회·진단 로직), C8(Webhook 통지·다중채널 이중화)은 API·데이터 명세가 명확하고  
  기능 범위가 한정적이어서 실습 난이도가 적정하다고 판단해 각 1표씩 분산함

### 케어 (제외: C8, C9)

- 비즈니스가치: C1 1표, C2 1표, C5 1표
- 실현가능성: C3 1표, C5 1표, C7 1표
- 사유: 비즈니스가치는 규제 시급성과 실수요 근거가 뚜렷한 후보에 분산함. C1은 알뜰폰 부정개통  
  비중 92.3%라는 정량 근거와 규제 준수 시급성이 가장 명확하고, C2는 2025년 사후규제 전환·60일  
  기한이라는 임박한 법적 압박이 뚜렷함. C5는 SKT가 통합조회 플랫폼을 운영하지 않는다는 As-Is가  
  구체적이고 요금폭탄 민원이라는 실수요가 설득력 있음. 실현가능성은 C3(NETCONF/REST 표준 API·  
  대시보드 구조 명확), C5(PCRF/PCF 단일 연동점으로 범위가 좁음), C7(정산 원장 대사 로직과 데이터  
  매핑이 구체적으로 정의됨)을 선택함

## 4. 선정 결과

### 최종 선정: C8. MVNO향 통신 장애·보안사고 실시간 통지 및 영향범위 조회 시스템

- **총점 8점으로 동점 없이 단독 1위** — 별도 동점 처리 절차(실현 가능성 우선·리서처 결정) 불필요
- 제안자 케어를 제외한 5명 중 4명(리서처·스토리텔러·데모·딜)이 비즈니스 가치 또는 실현 가능성  
  중 최소 한 축에서 C8에 투표함. 특히 데모는 두 사건(2025 SKT 유심유출·2018 KT 아현화재)의  
  실증 근거를 이유로 비즈니스가치 2표를 집중 배정함
- 2위 C5(7점) 대비 근소한 차이가 아닌 명확한 우위(1점차)이며, C8은 두 축 모두에서 고르게  
  득표(5/3)한 반면 C5는 실현가능성에 편중(3/4)됨

### 선정 주제 상세 (필드 스키마)

- **제안자**: 케어
- **발주 배경(WHY)**: W5(도매 용량·트래픽 가시성)를 장애·사고 통지 영역으로 확장한 실수요임.  
  현재 MNO의 대규모 망 장애·보안사고 발생 시 MVNO는 공식 통지를 실시간으로 받지 못하고  
  뉴스·SNS·자체 모니터링으로 뒤늦게 인지하는 구조임(As-Is). 2025년 4월 SKT 유심 정보 유출  
  사고 당시 KB리브엠 등 SKT망 알뜰폰 사업자 고객센터가 문의 폭주로 마비됐고, 2018년 KT  
  아현지사 화재 때도 알뜰폰 가입자 대상 보상 안내가 사고 발생 약 3주 후에야 개별 공지됨.  
  통지 지연이 MVNO 고객센터 마비와 가입자 이탈로 직결됨
- **업무 요구**
  - 장애·보안사고 발생 즉시 심각도별 자동 통지 (확인: 사고 인지 후 5분 이내 MVNO 담당자 통지)
  - 영향 가입자 규모·지역·서비스 범위 실시간 조회 (확인: 요청 후 3초 이내 응답, 표본 100건  
    100% 일치)
  - 복구 진행률·예상 복구시각 갱신 제공 (확인: 15분 주기 갱신, 복구완료까지 이력 연속 조회 가능)
  - 사고 종료 후 장애 이력·재발방지대책 리포트 자동 생성 (확인: 종료 후 3영업일 이내 제공)
- **기술 요구**
  - Webhook/API 기반 실시간 이벤트 푸시 (지표: 발생~통지 지연 1분 이내 / 성공률 99.9% / 월 측정)
  - 대규모 동시 조회 처리 300 TPS 이상 (측정조건: 장애 피크시간대, 응답 2초 이내)
  - 다중 채널 통지(API+이메일+SMS) 이중화, 단일 채널 장애 시 자동 전환 (지표: 전환 30초 이내)
  - 접근 인증 OAuth 2.0, 전송구간 암호화(TLS 1.2 이상), 통지 이력 위변조 방지 로그 보관
- **운용 요구**
  - 통지 시스템 자체 가용성 99.95% 이상 (연 다운타임 약 4시간 22분 이내)
  - 장애 통지 SLA 준수율 월 98% 이상, 미준수 시 페널티 조항 반영
  - 24x365 공동 대응 핫라인 운영 및 분기 1회 합동 장애대응 훈련(모의훈련) 실시
  - 월간 장애대응 리포트(통지 소요시간·MTTR·재발률) 제공
- **기대효과**: 장애 인지~통지 소요시간 평균 40분 → 5분 이내 단축, 장애 발생 시 MVNO 고객센터  
  문의량 30% 감소(예: 월 5,000건 → 3,500건)
