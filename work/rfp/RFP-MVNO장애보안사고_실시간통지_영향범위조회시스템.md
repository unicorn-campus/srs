### 약어표

- **MNO**: 망 보유 이동통신사 / **MVNO**: 망을 임차하는 알뜰폰 사업자  
- **NRC**: 초기 구축비(Non-Recurring Charge) / **MRC**: 월정 이용료(Monthly Recurring Charge) / **TCO**: 총소유비용  
- **NFR**: 비기능 요구사항 / **TPS**: 초당 처리건수 / **P99**: 상위 99퍼센타일(하위 99% 요청이 만족하는) 응답시간  
- **MTTR**: 평균 복구시간 / **ETR**: 예상 복구시각(Estimated Time to Restore) / **RTO/RPO**: 목표 복구시간/목표 복구시점  
- **HITL**: 사람 검수 개입(Human-in-the-Loop) / **PM**: 정기점검 / **CR**: 변경요청 / **SLA**: 서비스수준협약

## 0. 인사문

- 귀사의 무궁한 발전을 기원함. 당사 OO모바일(이하 "발주사")은 통신사(MNO) 망을 도매로 임차하여 알뜰폰
  서비스를 제공하는 MVNO 사업자로서, 망 장애·보안사고 발생 시 가입자를 신속·정확하게 보호하기 위한
  「통신 장애·보안사고 실시간 통지 및 영향범위 조회 시스템」 구축을 추진함.  
- 본 제안요청서(RFP)는 위 시스템 구축을 위한 요구사항·예산·일정·계약 조건을 정리하여 제안을 요청하는 문서임.
  귀사의 적극적인 제안을 요청함.  
- 명의: 발주사 최고정보책임자(CIO) / 사업총괄 부문장  
- 발행일: (가상) 2026-07-15

## 1. 사업 개요·추진 배경(취지)

### 1-1. 키 메시지 (WHY First)

- **"당사는 실시간 장애·사고 가시성을 확보하여, MNO 망 사고가 발생해도 가입자를 가장 먼저 안심시키는 알뜰폰
  사업자가 되고자 함."**

### 1-2. 발주사 개요·업계 동향

- 발주사는 MNO 망을 도매 임차하여 이동통신 재판매(알뜰폰) 서비스를 제공하는 MVNO임. 망 자산을 직접
  보유하지 않으므로 망 장애·보안사고에 대한 1차 정보는 MNO에 종속됨.  
- 업계 동향: 대규모 망 장애·개인정보 보안사고는 알뜰폰 사업자의 고객 신뢰·해지율에 직접적 영향을 미치며,
  통지 지연은 곧 고객센터 마비와 가입자 이탈로 연결됨.

### 1-3. 현재 문제점 (As-Is) — 장면화

- MNO의 대규모 망 장애·보안사고 발생 시, 발주사(MVNO)는 공식 통지를 실시간으로 받지 못하고 뉴스·SNS·
  자체 모니터링을 통해 뒤늦게 인지하는 구조임.  
- **당사 니어미스 사례**: 2025년 4월 MNO 유심(USIM) 정보 유출 사고 당시, 당사 고객센터에도 문의가 평소
  대비 수 배로 폭증했으나 공식 정보가 없어 상담사가 뉴스 속보를 보며 응대하는 상황이 발생함(해당 MNO 망
  기반 알뜰폰 고객센터 다수가 문의 폭주로 마비됨).  
- 2018년 MNO 국사(지사) 화재 당시, 알뜰폰 가입자 대상 보상 안내가 사고 발생 약 3주 후에야 개별 공지됨.  
- 결과적으로 "장애·사고 인지 지연 → 고객센터 응대 마비 → 가입자 이탈"의 악순환이 반복됨.

### 1-4. 정량 목표·기대효과 (현재값 → 목표값)

| 지표 | 현재값(As-Is) | 목표값(To-Be) | 성격 |
|------|---------------|----------------|------|
| 장애 인지 ~ MVNO 담당자 통지(평균) | 약 40분 | 5분 이내 | 내부 |
| **가입자 선제 안내 발송 시점** | 사후·수시간~수일 | **사고 인지 후 30분 이내** | **대가입자** |
| 장애 발생 시 고객센터 문의량(월) | 약 5,000건 | 3,500건(30% 감소) | 대가입자 |
| 영향범위(가입자·지역) 파악 방식 | 수작업·사후 추정 | 실시간 조회(P99 3초) | 내부 |

- **수미상관(인과 고리)**: 실시간 가시성 확보(WHY) → 담당자 **5분** 통지 → 검수된 안내 문구로 가입자
  **30분 내 선제 안내** → 문의 폭주 전 **가입자 안심**(문의 30% 감소). 헤드라인의 "가입자를 가장 먼저 안심"
  약속이 대가입자 지표로 닫힘.  
- **통지 목표 수치의 상·하위 관계(혼동 방지)**: "**5분**"은 인지~담당자 도달까지 운영 프로세스를 포함한 **종단
  목표**이고, 기술 요구의 "**1분**"(REQ-TEC-01)은 그 안에서 **시스템 전송 구간(MNO 이벤트 수신~시스템
  통지)** 목표로, 5분 목표의 부분집합임.

## 2. 요구사항

> 요구 분류: 업무(REQ-BIZ)·기술(REQ-TEC)·운용(REQ-OPS). 각 요구는 요구ID로 부록 A(RTM)와 연결됨.  
> 우선순위: 필수(M, Mandatory) / 권장(R, Recommended) / 선택(O, Optional).  
> 모든 요구에 측정 가능한 확인 기준(PoC 판정 기준)을 부여함.  
> **시각 기준 정의**: "발생/수신"=MNO 이벤트 수신(Webhook 수신) 시각, "인지"=수신 시각으로 정렬함.

|--------|:---:|-----------|----------------------|
| REQ-BIZ-01 | M | 장애·보안사고 발생 즉시 **심각도별 자동 통지** — MNO 이벤트를 심각도(긴급/높음/보통)로 구분하여 MVNO 담당자에게 자동 통지 | MNO 이벤트 수신(인지) 후 **5분 이내** MVNO 담당자 통지 도달(운영 프로세스 포함 종단 기준) |

| REQ-BIZ-05b | R | **AI 기반 고객센터 응대 지원** — 예상 문의 FAQ·응대 스크립트 초안 자동 생성 후 고객센터 제공 | 사고 통지 후 **10분 이내** 응대 스크립트 초안 생성, **상담사(HITL) 검수** 후 사용 |

- 엔드유저 상충 식별: 사내 운영자(고객센터·상황실)는 "정확·신속 통지와 응대 부담 경감"을, 경영진은 "가입자
  이탈 방지·신뢰 유지"를 원함. 상충 시 **가입자 보호(통지 신속성)**를 최우선 순위로 함.

**시스템 능력(Capacity) 수치표**

| 구분 | 지표 | 목표값 | 측정조건 |
|------|------|--------|----------|
| 성능 | 이벤트 통지 지연 | 1분 이내 | 수신~통지 종단, 월 측정 |
| 성능 | 영향범위 조회 응답 | 평균 2초 & P99 3초(동시 충족) | 피크 300 TPS 부하 시 |
| 용량 | 동시 조회 처리량 | 300 TPS 이상 | 장애 피크 시간대 |
| 용량 | 통지 대상 담당자 계정 | 500 계정 이상 | 개통 1년차 기준 |
| 용량 | 통지 이력 보관 | 3년 이상 | 위변조 방지 로그 포함 |
| 확장성 | 무중단 Scale-out | 지원 | 노드 증설 시 서비스 무중단 |

**장해 대책(가용성 → 이중화 → 배상 3단 연계)**

- **가용성 목표**: 통지 시스템 자체의 **서비스 중단 시간 기준** 가용성 **99.95% 이상/월**(월 허용 다운타임 약
  **21.6분**, 연 환산 약 4시간 22분). **이중화를 전제로 단일 컴포넌트 장애가 서비스 중단으로 전이되지 않음을
  가정**하며, 이 지표만 벤더 SLA로 함. → 2-3(운용)·6(계약)과 수치 일치.  
- **이중화**: 통지 채널 다중화(REQ-TEC-03), 네트워크·DB·DNS 이중화(REQ-TEC-06), 무중단 Scale-out.
  구체적 이중화 아키텍처(Active-Active/Standby)는 **벤더 제안에 위임**하되 위 가용성 목표 달성을 전제로 함.  
- **재해복구(DR)**: RTO 1시간·RPO 15분 목표(Capacity표), DR 전환 훈련 포함.  
- **MNO 망 복구시간(MTTR) 취급**: 실제 망 복구 주체는 MNO이므로 **MNO 복구 MTTR은 벤더 SLA에서 제외**
  하고, 월간 리포트의 **관찰 지표**로만 제공함. 벤더 책임은 복구정보(ETR) **15분 주기 갱신 성실성**임.  
- **배상**: 통지 시스템 가용성·통지 SLA 미달 시 서비스 크레딧(→ 6. 계약조건).

**보안·규제 준수**

## 3. 사업비·대가 산정 (얼마에)

> 본 예산은 교육용 가상 예제 수치이며, 제안 시 항목별 상세 견적 제출을 요구함.  
> 공개 전략: **총액 상한(band) 제시 + 항목별 명세 제출** 방식.

- **비용 구분(초기 NRC / 계속 MRC 분리 제출 필수)**

| 구분 | 항목 | 가상 예산 상한(band) |
|------|------|----------------------|
| 초기(NRC) | 설계·개발·MNO 연동·AI 모델 구축·데이터 이행·**DR·이중화 초기 구축**·테스트 | **8억원 이내** |
| 계속(MRC) | 회선·인프라 이용료, 유지관리, 헬프데스크, **DR·이중화 운영**, AI 모델 재학습·운영 | **월 2,500만원 이내** |
| TCO(5년) | **NRC + MRC×60개월** (DR·이중화 포함) | 약 **23억원**, 상한 **25억원 이내** |

- **예산 정합성**: NRC 8억 + MRC 2,500만×60 = 23억으로 TCO 상한 25억 이내에 수렴함(세 상한 동시 충족).  
- **별도 예산 항목 계상처 명시**: 데이터 이행·병행운영(NRC 내), DR·이중화(초기=NRC / 운영=MRC),
  AI 모델 재학습·성능관리(MRC 내), 하자보수 종료 후 유상 유지관리(별도), 변경요청(CR) 맨먼스 단가(별도 단가표).  
- **유지관리 요율**: 요율 **상한을 개발비의 12~15%**로 제시하며(또는 SW사업 대가산정 가이드 준거 산정 근거
  제출), 상용SW 포함 시 등급별 요율 근거를 첨부.  
- **실비정산 vs 정액 구분**: 출장·교통비, 회선 초과사용(over-usage), CR 실투입 M/M는 **실비정산**, 유지관리·운영은
  **정액**으로 구분 요구.  
- TCO 관점: HW·SW 획득비만이 아니라 운영·유지보수·다운타임 비용을 포함한 5년 생애주기 총비용으로 비교함.

## 6. 계약 조건 (SLA·NDA·저작권)

### 6-1. SLA·서비스 크레딧

- SLA 지표는 2-3(운용) SLA 지표표(벤더 통제 가능 지표)를 계약 기준선으로 준용함(수치 일치).  
- **서비스 크레딧 산정식(월별 적용·단조증가·인센티브 균형)**:

| 월 통지 SLA 준수율 | 서비스 크레딧(월 MRC 대비) |
|--------------------|-----------------------------|
| 98% 이상 | 없음(목표 달성) |
| 97% 이상 ~ 98% 미만 | 5% |
| 95% 이상 ~ 97% 미만 | 10% |
| 93% 이상 ~ 95% 미만 | 15% |
| 93% 미만 | 20% (월 상한) |

- **적용 규칙**: 크레딧은 **월별 산정·적용**하며, **연간 누적 크레딧 상한은 연 MRC의 15%**로 함. 크레딧은
  징벌이 아닌 개선 유인으로 설계하며, 기대목표치(98%)/최소목표치(93%)를 이원화함.

### 6-2. 저작권

- **분리 귀속**: 신규 개발 산출물은 발주사 귀속, 벤더 기존 보유자산(선행 저작물·AI 사전학습 모델 등)은 벤더
  귀속 + 발주사에 사용권 부여.

### 6-3. 하자보수 vs 유지관리

- **하자보수(무상)**: 사업 완료일로부터 **1년**, "오류·결함 수정"에 한정.  
- **유지관리(유상)**: 추가개발·기능개선·추가 교육·환경변경 등은 유상 유지관리(MRC/별도)로 구분.  
- **AI 재학습 귀속 경계(분쟁 예방)**: 정상 운영 중 모델 성능저하(drift)로 인한 **정기 재학습은 MRC 유상**,
  명백한 **구현결함 기인 성능오류는 무상 하자**로 구분함. 판정 기준은 **REQ-BIZ-05a 백테스트 오차율(±15%)
  초과 원인이 구현결함인지 여부**로 함.

### 6-4. 지체상금

- 요율: SW사업(물품+용역 일괄) 기준 **1000분의 1.5**. 산식 = **기준금액 × 요율 × 지체일수**.  
- **기준금액**: 계약금액(NRC) 중 **미완료·미검수 부분**으로 한정(이미 검수·기성지급된 부분 제외).  
- **총액 상한(cap)**: 지체상금 총액은 **계약금액(NRC)의 30%**를 초과하지 않음.  
- 면책: 발주사 귀책(요구 확정·검수 지연)·불가항력.

### 6-5. NDA·규제 준수

- **NDA 2회**: RFP 배포 시점(제안 참여 벤더 전원) + 계약 체결 시점. 비밀정보 범위·목적 외 사용금지·제3자
  공개금지·반환/파기·위반 시 손해배상.  
- **규제 준수 서약**: 개인정보보호법 안전성 확보조치, 연동 구간 보안 준수 의무를 NDA와 함께 명시.
