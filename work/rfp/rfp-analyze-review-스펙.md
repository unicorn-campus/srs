# RFP분석서 초안 리뷰 — 스펙(요구사항 정의 전문가)

> 리뷰어: 스펙 / 관점: 요구 정형화 수준·SRS(05단계) 연계 가능성(S6 인수기준·S7 추적성)  
> 리뷰 대상: `work/rfp/rfp-analyze-draft.md`  
> 대사 근거: 확정 RFP `work/rfp/RFP-MVNO장애보안사고_실시간통지_영향범위조회시스템.md`,  
> SRS 가이드 `work/srs/srs-guide.md`(S3 FR·S4 NFR·S6 AC·S7 RTM)  
> 판정 원칙: "측정할 수 없으면 요구사항이 아님" · RFP↔SRS↔AC 추적성(ISO/IEC/IEEE 29148)

---

## 1. 총평

16개 RFP 요구ID 전건을 표면요구·실제니즈·우선순위·대응가능여부 4열로 정형화하였고, 표면요구 열의  
정량치가 RFP·SRS 가이드 수치와 정합하여 FR/NFR·인수기준(AC) 이식 가능성이 높음.  
우선순위 판정(필수 13·권장 3)이 RFP 부록A의 M/R과 100% 일치하여 S7 RTM 우선순위 열 정합성이 우수함.  
다만 자기선언 범위("2장 전 요구") 대비 REQ-ID 외 측정 가능 요구(Capacity/DR NFR·교육·이행 요건)가  
분석에서 빠져 SRS S4/S7 입력에 공백이 있으므로 이 1건은 수정이 필요함.

---

## 2. 지적사항

### 2-1. 수정 필수

#### [필수-1] REQ-ID 외 측정 가능 요구 미분석 — SRS S4/S7 입력 공백

- **위치**: A2·A3 요구별 표 전체 / A4 "RFP 커버리지 대사"(누락 0건) / A0 범위("2장 전 요구")  
- **문제**:  
  - RFP 2-2 Capacity 수치표의 측정 가능 요구 — 통지 담당자 계정 **500 계정 이상**, 통지 이력 보관  
    **3년 이상**, 무중단 **Scale-out**, DR **RTO 1시간/RPO 15분** — 이 16개 REQ-ID 분석 어디에도  
    표면요구·대응가능여부로 등장하지 않음.  
  - RFP 2-3 하단 이행 요건 — 교육 대상 3종(상담사·상황실·관리자), 3단계 지식이전(Shadow→분담→독립),  
    hypercare **1개월 SLA**, 운영 런북, 롤백 계획 — 역시 REQ-OPS-01~04 어디에도 분석되지 않음.  
  - SRS 가이드는 이들을 S4 **NFR-C02/C03/A03/A04**(근거: Capacity표) 및 S7 운영이행 요건으로 별도  
    명세함. 따라서 초안의 "누락 0건"은 REQ-ID 기준일 뿐이며, 자기선언 범위(2장 전 요구)와 상충함.  
  - 특히 DR RTO 1시간·RPO 15분·3년 보관·500 계정의 **대응가능여부(기술 실현성·비용)가 미검증** 상태로  
    SRS S4/S7에 넘어가, 분석서 목적("SRS 작성의 검증된 입력자료 제공", A0)이 부분 미달됨.  
- **개선 제안**:  
  - Capacity/DR NFR을 상위 REQ-TEC-06(인프라·이중화)·REQ-OPS-01(가용성) 하위 세부 확인기준으로  
    편입하거나 별도 행을 추가하고, 각 항목에 대응가능여부(가능/부분가능+근거)를 부여함.  
  - 교육·이행(전환) 요건을 REQ-OPS-03/04 연계 또는 별도 "이행 요구"로 분석하여 S7 운영이행에 연결함.  
  - A4 "누락 0건" 문구를 "REQ-ID 기준 누락 0건, Capacity/DR·이행 세부 요구는 상위 요구에 편입"으로 정정함.

### 2-2. 권장

#### [권장-1] REQ-BIZ-05a 인수기준 성격(측정·리포트형) 소실 — 과약속 위험

- **위치**: A1 REQ-BIZ-05a 표면요구 열 / A4 핵심발견 4  
- **문제**: 표면요구를 "백테스트 오차율 ±15%"로 압축함. RFP 원문(2-1)은 "건별 오차율 ±15% 이내를  
  합격으로 하며 **합격 건수를 리포트**"로, 임계 합격률(SLA)이 아닌 **측정·리포트형** 기준임. 압축 표현은  
  SRS S6에서 합격률 임계값(예: ≥90%)을 임의 신설하여 RFP 이상으로 과약속할 위험을 유발함. 또한 ±15%는  
  RFP 6-3 하자/유지 귀속 경계 판정 기준이기도 함.  
- **개선 제안**: 05a의 AC 성격을 "측정·리포트형(비-SLA 임계), ±15%는 하자/유지 경계(6-3) 귀속 판정  
  기준"으로 명시함. A4 "확률형 인수 기준"에 "합격률 임계 임의 신설 금지·합격 건수 리포트" 단서를 추가함.

#### [권장-2] 통지 SLA 준수율(REQ-OPS-02)의 N:1 집계 관계 미명시

- **위치**: A3 REQ-OPS-02 / A2 REQ-TEC-01·03 / A1 REQ-BIZ-03  
- **문제**: RFP 2-3 SLA 지표표상 통지 준수율(REQ-OPS-02)은 수신~통지 1분(REQ-TEC-01)·채널 자동 전환  
  30초(REQ-TEC-03)·ETR 15분 갱신 성실성(REQ-BIZ-03) 미달을 함께 "통지 준수율에 반영"하는 **다대일(N:1)  
  집계 지표**임. 분석서는 각 요구를 독립 행으로만 판정하여 이 집계 관계를 드러내지 않음. SRS S6 준수율  
  산식(준수건/전체건)에서 "위반 1건"의 판정 기준(어느 하위 지표 미달이 위반인지)이 모호해질 소지가 있음.  
- **개선 제안**: REQ-OPS-02 행에 "준수율 산입 하위 지표 = REQ-TEC-01·REQ-TEC-03·REQ-BIZ-03"을 명시함.  
  S6 산식에서 "준수 건" 정의를 하위 지표 동시 충족 기준으로 못 박도록 연계 메모를 부기함.

#### [권장-3] REQ-BIZ-03 이중 AC(기능+SLA 준수율 98%) 미분리

- **위치**: A1 REQ-BIZ-03  
- **문제**: REQ-BIZ-03은 기능 AC(15분 주기 갱신·완료까지 연속 조회)와 SLA AC(ETR 갱신 성실성 준수율  
  **≥98%**, RFP 2-3·6-1)를 동시에 파생함. 표면요구 열에는 기능 AC만 있고 98% 준수율은 우선순위 근거  
  ("통지 준수율 반영")에만 암시됨. SRS S6/S7에서 두 AC가 한 요구ID에 묶여 추적 명료성이 저하됨.  
- **개선 제안**: REQ-BIZ-03에 기능 AC와 SLA AC(98%)를 분리 표기하여, SRS FR-ETR-01(기능)·SLA 지표  
  (갱신 성실성 준수율)로의 이중 매핑을 사전 예고함.

#### [권장-4] 부분가능 3건 조건의 SRS 가정/제약 태깅 부재

- **위치**: A1 REQ-BIZ-05a·05b / A2 REQ-TEC-05 대응가능여부 열  
- **문제**: 부분가능 3건의 조건(과거 사고 데이터 확보량·품질, HITL 검수 전제, MLOps 재학습 파이프라인  
  구축 전제)은 SRS S3 가정(Assumptions)·제약(Constraints)으로 이관되어야 하나, 분석서는 대응가능여부  
  근거 문장으로만 서술하여 SRS 편입 슬롯이 지정되지 않음(실체는 S3 예시와 일치, 라벨링만 미비).  
- **개선 제안**: 각 조건에 "→ SRS 가정" 또는 "→ SRS 제약" 태그를 부기하여 S3 4구분(범위·제외·가정·제약)  
  이식을 예고함. 참고용·HITL은 제약, 데이터 확보량·MNO 실시간 제공은 가정으로 분류함.

### 2-3. 기타 경미(참고, 건수 미포함)

- **PRB 참조 일관성**: REQ-BIZ-02·04만 PRB-03·04를 인용하고 01·03·05 등은 미인용함. PRB는 SRS S2  
  예시 ID이므로 RFP분석서가 특정 PRB 번호를 인용하면 S2 재번호 시 추적 단절 소지가 있음. 인용을 전건  
  일관 적용하거나 As-Is 서술 참조(RFP 1-3)로 통일함을 권장함.

## 3. SRS 연계 — 강점·보완점

### 3-1. 강점

- **정량성**: 표면요구 열의 임계값(5분·1분·전송성공률 99.9%·P99 3초·300 TPS·99.95%/21.6분·±15%·30초  
  등)이 RFP·가이드 수치와 정합함 → S3 FR·S4 NFR·S6 AC 임계값 이식이 용이함.  
- **우선순위 정합**: 필수 13·권장 3이 RFP 부록A의 M 13·R 3과 100% 일치함 → S7 RTM 우선순위 열이  
  무충돌로 확정됨.  
- **핵심발견의 사전 근거화**: A4 5건이 SRS 상류 근거를 제공함 — (5분⊃1분)→S6 지표 분리·S7 용어,  
  (MNO 종속)→S3 가정, (AI 참고용·HITL·룰폴백)→S3 제약·S6 확률형 AC, (벤더 SLA 경계·MNO MTTR 배제)  
  →S6 크레딧 대상 한정.  
- **추적 원칙 준수**: "원문 ID 승계·신규 ID 임의 생성 금지"를 명문화함 → RTM 후방 추적성(고아 항목 방지)  
  기반을 확보함.

### 3-2. 보완점

- (필수) REQ-ID 외 측정 가능 요구(Capacity/DR NFR·교육·이행 요건) 편입 — S4/S7 입력 공백 해소가 최우선임.  
- (권장) AC 성격(05a)·N:1 집계(OPS-02)·이중 AC(BIZ-03) 정밀화 및 부분가능 조건의 가정/제약 태깅으로  
  S3/S6/S7 핸드오프 정밀도를 높임.

## 4. 결론

- **수정 필수 1건**(필수-1: REQ-ID 외 측정 요구 미분석) · **권장 4건**(05a AC 성격, OPS-02 N:1 집계,  
  BIZ-03 이중 AC, 부분가능 가정/제약 태깅) · 경미 참고 1건(PRB 참조 일관성).  
- 초안은 정량성·우선순위 정합·추적 원칙 준수 측면에서 전반 품질이 우수함. **필수-1(측정 요구 공백)만  
  보완하면** SRS 05단계 작성의 검증된 입력자료로 적합함. 권장 4건은 SRS 정형화 정밀도 향상용으로 반영을  
  권함.
