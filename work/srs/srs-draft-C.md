## 6. 핵심 처리·판정 대상 요구사항

> 시스템이 무엇을 근거로 처리·판정하는지 유형별(6.1 ~ 6.7)로 명세함. 각 판정은 합격/등급을
> 가르는 측정 가능한 임계값을 동반하며, 결과 유형을 정책(7장)·상태값(11장)과 연결함.
> AI 판정은 확률형(오차율·표본·룰 폴백·HITL)으로 서술하여 결정론적 합격을 과약속하지 않음.

### 6.1 심각도 판정

- 판정 입력: 사고유형(장애/보안) · 영향 국사/셀 규모 · 영향 서비스 범위 · 과거 유사사고 이력.
- 판정 내용: 사고유형·영향 규모·영향 서비스 조합으로 **긴급/높음/보통** 3등급 자동 분류.
- 측정 임계값: 심각도 분류 규칙표 대사 표본 100건에 대해 **100% 일치**(판정 로직 결정론적).
- 결과 연계: 6.2 통지 대상·채널 판정 입력, 7.2 심각도·채널 정책, 16장 반복 수신 상향 재판정. [REQ-BIZ-01]

### 6.2 통지 대상·채널 판정(5분 종단)

- 판정 입력: 심각도(6.1) · 담당자 등급 · 소속 팀 · 시간대 · 채널 우선순위 · 수신 가능 시간대.
- 판정 내용: 통지 대상·채널·발송 순서 결정 후 다중 채널 발송, 단일 채널 장애 시 대체 채널 자동 전환.
- 측정 임계값(2종 분리): **인지 ~ 담당자 도달 5분 이내(종단)** ⊃ **시스템 수신 ~ 통지 1분 이내(전송)**.
  두 지표는 상·하위 부분집합 관계로 별도 집계함(혼동 시 정산 분쟁 소지 — A4-2 근거).
- 부수 임계값: 전송 성공률 **99.9% 이상**[REQ-TEC-01], 단일 채널 장애 시 **30초 이내** 자동 전환[REQ-TEC-03].
- 결과 연계: 11장 통지 지연·인지~담당자 초과·통지 실패 상태. [REQ-BIZ-01, REQ-TEC-01, REQ-TEC-03]

### 6.3 영향범위 판정(P99 3초)

- 판정 입력: 자사 가입자 분포(지역·서비스·요금제 집계 차원) × MNO 영향정보(영향 국사/셀·서비스 범위) 조인.
- 판정 내용: 영향 가입자 규모·지역·서비스 범위 실시간 산출(집계 단위, 개인 식별자 미포함).
- 측정 임계값: 피크 **300 TPS**에서 **평균 2초 & P99 3초**(동시 충족), 표본 100건 대사 **100% 일치**.
- 결과 연계: 11장 조회 지연 상태, 16장 소규모 집계 개인 특정 차단(범위 상향 집계). [REQ-BIZ-02, REQ-TEC-02]

### 6.4 AI 영향예측 판정(확률형)

- 판정 입력: 과거 유사사고 이력 · 현재 사고 컨텍스트 · 영향범위 집계(6.3) · 예측 신뢰도.
- 판정 내용: 과거 유사사고 패턴으로 영향 규모·확산 추이 예측(참고용, 의사결정 보조).
- 측정 임계값: 백테스트 **오차율 ±15% 이내** 건만 합격 집계·리포트 반영, **추론 3초 이내**.
- 확률형 조건: 표본 부족 시 **룰 기반 폴백 병행**, 결과는 참고용으로 **담당자 HITL 검수** 필수(무검수 확정 금지).
- 결과 연계: 오차율 ±15% 초과 시 11장 "추가 검토 필요"로 휴먼 검토 이관, 16장 신규 유형 폴백 병행.
  [REQ-BIZ-05a, REQ-TEC-05]

### 6.5 AI 응대지원 판정(확률형)

- 판정 입력: 사고 심각도·영향범위·예상 문의 유형 · FAQ 데이터 · 통지 내용.
- 판정 내용: 예상 문의 FAQ·응대 스크립트 초안 생성(상담사 응대 부담 경감, 참고용).
- 측정 임계값: **통지 후 10분 이내** 초안 생성, **추론 3초 이내**.
- 확률형 조건: 생성 결과는 **상담사 HITL 검수** 후 사용(무검수 자동 발송 금지).
- 결과 연계: 10분 초과 시 11장 "응대 지연" 상태. [REQ-BIZ-05b, REQ-TEC-05]

### 6.6 ETR 갱신 판정(15분 주기)

- 판정 입력: MNO 수신 복구정보(복구 진행률·예상 복구시각) · 사고 진행 상태 · 원천 데이터 적시성.
- 판정 내용: 복구 진행률·예상 복구시각(ETR)을 산출·갱신하여 조회·통지에 반영.
- 측정 임계값: 사고 완료까지 **15분 주기 연속 갱신**(원천 수신 시 배치/스트림 갱신).
- 면책 조건: 원천 복구정보가 MNO 미제공이면 미갱신을 면책 처리하고 성실성 로그로 입증(11장·16장 연계).
  [REQ-BIZ-03]

### 6.7 사고 리포트 생성 판정(3영업일)

- 판정 입력: 사고 종료 판정 · 통지 이력 · 영향범위 산출 이력 · MNO 복구 MTTR(관찰지표) · 타임라인.
- 판정 내용: 장애 이력·재발방지 리포트(타임라인·영향범위·MTTR) 자동 생성.
- 측정 임계값: 사고 종료 후 **3영업일 이내** 생성.
- 결과 연계: 3영업일 초과 시 11장 "리포트 지연" 상태. [REQ-BIZ-04]

## 7. 정책 요구사항

> 동일 이벤트라도 대상·조건에 따라 다른 처리 정책을 적용함(7.1 ~ 7.5). 정책은 2장 기본 원칙을
> 하위 규칙으로 구체화하며, 충돌 시 적용 우선순위(9.2 연계)를 따름. 조건부 추가 판정으로
> 무조건 통과를 방지함.

### 7.1 담당자 등급 정책

- 신규 담당자(온보딩 초기 등급)는 통지 확인·에스컬레이션을 강화하고, 숙련 담당자는 표준 정책을 적용함.
- 담당자 등급별 통지 준수 이력을 반영하여 준수율(월 98%) 미달 위험 담당자에 강화 정책을 적용함.
  [REQ-BIZ-01, REQ-OPS-02]

### 7.2 심각도·채널 정책

- **긴급**: 전 채널 동시 발송 + 슈퍼바이저 즉시 알림.
- **높음**: 주 채널 발송 + 백업 채널 대기.
- **보통**: 주 채널 발송.
- 채널 장애 시 30초 이내 대체 채널 자동 전환(6.2 연계). [REQ-BIZ-01, REQ-TEC-03]

### 7.3 시간대·구간 정책(야간·대량 클레임)

- 야간·대량 클레임 구간은 통지 우선순위를 상향하고 지연 판정을 억제함.
- 대량 클레임 구간은 일부 대상 우선 통지로 5분 종단 SLA 침범을 방지함(전환 규칙은 10.2). [REQ-BIZ-01, REQ-OPS-02]

### 7.4 조건부 추가 판정 정책(무조건 통과 방지)

- 통지 억제 대상이라도 **신규 사고유형 · 심각도 급변 · 과거 클레임 고객 다수 포함** 시 추가 판정을 수행함.
- 억제 대상의 무조건 통과를 금지하고, 추가 판정 결과를 감사 로그에 기록함(16장 예외 연계). [REQ-OPS-02]

### 7.5 정책 우선순위(상위 위반 정책 우선)

- 정책 충돌 시 **상위 위반·규제 정책이 하위 허용 정책에 우선**함(9.2 우선순위 단계 준거).
- 상위 위반 정책에 해당하면 허용룰이 있어도 억제·보류하여 규제·SLA 리스크를 통제함. [REQ-OPS-02]

## 8. 특수·강화 정책

> 리스크가 큰 특정 조건에서 표준 정책보다 강화된 처리를 적용함. 본 장은 **강화 트리거·강화 내용**을
> 규정하며, 전환 규칙·지연 상태는 10장에서 분리 규정함(경계 분리).

- **신규 사고유형 트리거**(전체/팀/담당자 최초 발생): 실시간 판정 필수 + 슈퍼바이저 우선 알림 + HITL 검토
  우선 배정. [REQ-TEC-06]
- **대량 통지 트리거**(동시 통지 대상 임계 초과): 일부 대상 우선 통지 + 통지 성실성 기록 강화(구체 전환·지연
  리포팅 규칙은 10.2). [REQ-BIZ-01]
- **위험등급 초과 트리거**(위험 등급 임계 초과): 즉시 에스컬레이션 + 휴먼(HITL) 검토 우선 배정. [REQ-BIZ-01]
- **MNO 원천 지연 트리거**(원천 데이터 적시성 저하 구간): 통지 성실성 기록을 강화하여 사후 면책 근거를 확보하고,
  채널 자동 전환(30초)·병렬 전송을 유지함(A4-3 MNO 종속 근거). [REQ-TEC-03, REQ-TEC-06]

## 9. 룰/리스트 관리 요구사항

> 허용/억제 룰의 적용 단위·우선순위·등록·이력을 명세함. 적용 단위는 표(9.1), 우선순위는 상위→하위
> 다단계 나열(9.2)로 표현함. 상위 위반·규제 룰이 하위 허용 룰에 우선함을 명문화함.

### 9.1 룰 적용 단위(표)

| 적용 단위 | 설명 |
|-----------|------|
| 전체 공통 | 모든 팀·채널에 공통 적용 |
| 부문/팀 | 부문·팀ID 기준 적용 |
| 담당자 | 담당자 사번 기준 적용 |
| 사고유형 | 장애/보안 사고유형 기준 적용 |
| 채널 | 통지 채널(SMS/이메일/API) 기준 적용 |
| 조합 | 팀 + 사고유형 + 채널 조합 기준 적용 |

### 9.2 우선순위(다단계 나열, 상위→하위)

- 규제·법적 즉시 대응
  - 확정 억제(BlackList)
    - 긴급 운영 차단
      - 대상 제재 상태
        - 조합 허용룰
          - 사고유형 허용룰
            - 팀 허용룰
              - 담당자 허용룰
                - 전체 공통 허용룰
                  - 신규 사고 정책
                    - 일반 룰 판정
                      - 모니터링 정책
- 상위 위반 정책에 해당하면 하위 허용룰이 있어도 억제·보류함(7.5 정책 우선순위 준거).

### 9.3 등록·승인

- 필수 기록 항목: 등록 단위 · 대상 · 적용 기간 · 사유 · 요청자 · 승인자 · 변경 이력.
- 적용 방식 지원: **즉시 / 예약 / 임시 / 만료** 지정 적용.
- 억제룰 등록 시 대상·사유·승인자를 필수 입력하고, 신규 사고유형 예외(7.4·16장)를 함께 검토함. [REQ-TEC-04]

### 9.4 관리·이력

- 룰 등록·수정·해제는 전량 감사 로그로 남김(13장 감사 로그 연계). [REQ-TEC-04]
- **준수율 분모 처리 규칙**: 억제룰로 의도적 미통지된 건은 통지 SLA 준수율(월 98%) 산정 **분모에서 제외**하며,
  제외 근거(적용 룰ID·사유·승인자)를 감사 로그로 입증함. 억제룰 부재 미통지는 분모에 포함하여 미준수로 집계함.
  [REQ-OPS-02]

## 10. 신규·예외 흐름 및 지연 처리 요구사항

> 표준 즉시 처리로 확정하기 어려운 신규·고위험 이벤트의 처리 흐름과 지연 판정 전환·상태를 명세함.
> 8장(강화 트리거·내용)과 분리하여 본 장은 **전환 규칙·지연 상태값**을 규정함.

### 10.1 신규 판단 기준

- 전체 최초 발생 사고유형 · 팀 최초 발생 · 담당자 최초 발생.
- 기존 사고유형의 임계 초과 변형(영향 규모·확산 속도 급변).
- MNO 원천 규격(페이로드·인터페이스) 변경. [REQ-TEC-06]

### 10.2 실시간/지연 전환

- 신규·저신뢰 예측 건은 **실시간 판정 필수** + 일부 대상 **우선 통지**, 나머지는 **지연 리포팅**으로 전환함.
- 지연 전환 건은 **AI 예측·휴먼 검토를 병행**한 뒤 운영자가 최종 확정함(6.4 확률형·HITL 연계).
- 대량 통지 구간은 우선 통지로 5분 종단 SLA를 보호하고, 나머지는 지연 큐로 이관함(7.3·8장 연계).
  [REQ-TEC-01, REQ-TEC-03]

### 10.3 지연 상태값

- **지연 대기** / **지연 진행 중** / **지연 완료** / **지연 실패(재시도 필요)** 4단계로 관리함.
- **24시간 이내 미완료** 시 자동으로 슈퍼바이저에게 알림을 발송함(16장 예외 연계). [REQ-OPS-02]

## 11. 처리 결과 상태값

> 처리 결과를 다단계 상태값으로 세분하고 표(상태/정의/조건/후속 조치)로 표현함. 각 상태의 진입
> 조건은 측정 가능한 임계값·건수·시간으로 서술함. AI 저신뢰·SLA 미충족·가용성 미달·면책 상태를
> 포함함(16장 예외 케이스 연계).

| 상태 | 정의 | 조건(측정 가능) | 후속 조치 |
|------|------|-----------------|-----------|
| 정상 통지 | 모든 통지 정책 준수 | 위반·지연 없음(1분·5분·성공률 99.9% 충족) | 별도 조치 없음 |

## 16. 예외·보완 케이스

> 표준 규칙만으로 판단이 어긋날 수 있는 엣지 케이스를 X-RAY 스타일("~하더라도 ~인 경우 ~함")로
> 나열함. 각 케이스는 [일반 규칙 성립] + [예외 조건] + [예외 처리] 3요소를 갖추며, 2장 원칙·7장
> 정책·11장 상태값과 정합함. 벤더 면책 경계를 명문화하여 정산 분쟁을 예방함.

- 통지 억제룰 등록 대상이더라도, **신규 사고유형**이면 무조건 통과를 금지하고 추가 판정을 수행함.
  [REQ-OPS-02, 7.4]
- 통지 SLA(수신 ~ 통지 1분)를 충족하더라도, **원천 데이터가 MNO 미제공으로 지연**된 경우 벤더 면책으로
  분류하고 성실성 로그로 입증함. [REQ-BIZ-03, A4-3·5]
- AI 영향예측 신뢰도가 임계(오차율 ±15%)를 초과하더라도, **신규 사고유형으로 표본이 부족**하면 룰 기반
  폴백을 병행함. [REQ-BIZ-05a, REQ-TEC-05]
- 시스템 전송 SLA(수신 ~ 통지 1분)를 충족하더라도, **인지 ~ 담당자 5분 종단을 초과**하면 지연으로 분류함
  (1분 ⊂ 5분 부분집합 관계 — A4-2). [REQ-BIZ-01, REQ-TEC-01]
- 영향범위 조회가 가능하더라도, **소규모 집계로 개인 특정이 가능**한 구간은 범위를 상향 집계하여 개인 특정을
  차단함. [REQ-BIZ-02, REQ-TEC-04]
- 지연 판정으로 전환되었더라도, **24시간 이내 완료되지 않으면** 자동으로 슈퍼바이저에게 알림을 발송함.
  [REQ-OPS-02, 10.3]
- 심각도가 보통으로 분류되었더라도, **동일 사고가 임계 시간 내 반복 수신**되면 심각도를 상향 재판정함.
  [REQ-BIZ-01, 6.1]
