# SRS 초안 — SEC3. 제안 솔루션 기능 요구사항(FR)·범위

> 담당: 스펙(작성) / 아키 리뷰. 근거: RFP분석서 `work/rfp/rfp-analyze.md`(대응가능 "가능·부분가능"  
> 요구), SRS 가이드 S3. 기능ID는 가이드 S3 체계(`FR-{그룹}-{번호}`)를 승계하며, 각 FR에 근거  
> RFP 요구ID를 1:1 매핑함(신규 요구ID 생성 금지). 상세 인수 기준은 SEC5, 추적성은 SEC6 참조.

## 3.1 솔루션 범위

- **범위(In-scope)**: MNO 장애·보안사고 이벤트 수신→심각도별 자동 통지, 영향범위 실시간 조회,  
  복구정보(ETR) 갱신, 사고 리포트 자동생성, AI 영향예측·응대지원, 24x365 공동대응 운영·월간  
  운영 리포트. RFP 요구ID 16건 전부를 기능(FR)·비기능(SEC4 NFR)·인수기준(SEC5)으로 명세함.  
- **제외범위(Out-of-scope)**: ① MNO 망 자체의 물리적 복구(벤더 SLA 제외, 관찰 지표만) ② 가입자  
  개인 식별정보 원문 제공(집계·범위 단위만) ③ AI 결과의 무검수 자동 확정·발송(HITL 필수)  
  ④ 이중화 아키텍처·배포형태 고정(가용성 목표 전제로 벤더 제안 위임).  
- **가정(Assumptions)**: MNO가 이벤트를 Webhook/API 규격으로 실시간 제공(REQ-TEC-01), AI 백테스트용  
  과거 사고 데이터 확보 가능(부분가능 요구 REQ-BIZ-05a·05b·REQ-TEC-05의 성능 전제).  
- **제약(Constraints)**: 개인정보보호법 안전성 확보조치 준수, 접근권한 3단계(관리자/운영자/조회자)  
  분리, AI 결과는 참고용·HITL 게이트 통과 필수.

| FR-통지-02 | 심각도(긴급/높음/보통) 자동 분류 | REQ-BIZ-01 | 필수 | 가능 |

| FR-조회-02 | 집계·범위 단위 제공(개인식별 최소화) | REQ-BIZ-02 | 필수 | 가능 |

  준수율)는 비기능 요건으로 SEC4(NFR)·SEC5(인수기준)에서 명세하며 SEC6 RTM에서 교차 연결함.

## 3.3 기능 상세 (ISO/IEC/IEEE 29148 5요소: 조건+주체+동작+대상+제약)

- **FR-통지-01 (심각도별 자동 통지)**: MNO 장애·보안 이벤트를 수신하면, 시스템은 심각도별로 해당  
  MVNO 담당자에게 통지를 발송함 — 수신(인지)~담당자 도달 5분 이내(운영 포함 종단). [REQ-BIZ-01]  
- **FR-통지-02 (심각도 자동 분류)**: 이벤트 수신 시, 시스템은 사고유형·영향 규모를 규칙엔진으로  
  긴급/높음/보통으로 분류함 — 분류 규칙 대사 100% 정확. [REQ-BIZ-01]  
- **FR-조회-01 (영향범위 실시간 조회)**: 운영자가 조회를 요청하면, 시스템은 영향 가입자 규모·지역·  
  서비스 범위를 실시간 반환함 — 평균 2초 & P99 3초(피크 300 TPS), 표본 100건 대사 100% 일치.  
  [REQ-BIZ-02]  
- **FR-조회-02 (집계·범위 단위 제공)**: 조회 응답 시, 시스템은 개인 식별자 원문을 배제하고 집계·  
  범위 단위로만 제공함 — 개인 특정 불가(개인정보보호법 안전성 확보조치). [REQ-BIZ-02]  
- **FR-ETR-01 (복구정보 갱신)**: 사고 진행 중, 시스템은 복구 진행률·예상 복구시각(ETR)을 15분  
  주기로 갱신하고 완료 시점까지 연속 조회를 제공함. [REQ-BIZ-03]  
- **FR-리포트-01 (사고 리포트 자동생성)**: 사고 종료 시, 시스템은 통지 타임라인·영향범위·MTTR을  
  포함한 장애 이력·재발방지 리포트를 자동 생성함 — 종료 후 3영업일 이내. [REQ-BIZ-04]  
- **FR-AI예측-01 (AI 영향예측)**: 사고 이벤트 입력 시, 시스템은 과거 유사사고 패턴으로 영향 가입자  
  규모·확산 추이를 예측함 — 백테스트 오차율 ±15% 이내 건을 합격 집계·리포트, 표본 부족 시 룰 기반  
  폴백 병행, 추론 3초 이내, 신규 사고 누적 10건 시 재학습·분기 성능점검. 결과는 참고용(HITL).  
  [REQ-BIZ-05a, REQ-TEC-05]  
- **FR-AI응대-01 (AI 응대지원)**: 사고 통지 후, 시스템은 예상 문의 FAQ·응대 스크립트 초안을  
  생성하여 고객센터에 제공함 — 통지 후 10분 이내 초안, 상담사(HITL) 검수 후 사용, 추론 3초.  
  [REQ-BIZ-05b, REQ-TEC-05]  
- **FR-운영-01 (공동대응 운영)**: 시스템·운영조직은 24x365 공동대응 핫라인을 운영하고 분기 1회  
  합동 장애대응 모의훈련을 실시·기록함 — 훈련 결과 리포트 제출. [REQ-OPS-03]  
- **FR-리포트-02 (월간 운영 리포트)**: 매월, 시스템은 통지 소요시간·MNO 복구 MTTR(관찰 지표)·  
  재발률을 포함한 월간 운영 리포트를 자동 생성함 — 정기 리뷰 미팅 근거. [REQ-OPS-04]

- 각 FR의 합격/불합격 판정 임계값은 SEC5(인수 기준)에 측정 가능한 형태로 명세하며, AI 기능  
  (FR-AI*)은 확률형 판정(오차율·표본·룰 폴백·HITL)으로 서술함.
