# SRS 초안 — SEC5. 인수 기준(Acceptance Criteria)

> 담당: 스펙(작성) / 딜 리뷰(정산·SLA 리스크). 근거: RFP분석서 A4 계약·가격 리스크, SRS 가이드 S6.  
> 원칙: "측정할 수 없으면 요구사항이 아님" — 모든 인수 기준은 [지표+임계값+측정조건·판정방법]으로  
> 종결하고 모호어를 배제함. 각 인수 기준은 RFP 요구ID·FR/NFR과 1:1 추적(SEC6 RTM).

## 5.1 인수 기준 작성 규칙

- 서술 형식은 [지표 + 임계값 + 측정조건·판정방법]을 기본으로 하며, 필요 시 Given-When-Then으로  
  선행조건·행위·관찰 결과를 명시함. Then 절은 반드시 임계값·시간·건수로 종결함.  
- 측정 4요소를 명시함: ① 지표 정의(산식) ② 임계값 ③ 측정 방법·데이터 원천(자동 타임스탬프·로그)  
  ④ 측정 주기·집계 단위(월·건별). 벤더 통제 가능 지표만 크레딧·페널티 대상으로 함.  
- AI 기능은 확률형 판정(오차율·표본·룰 폴백·HITL)으로 서술하고 결정론적 합격률을 과약속하지 않음.

## 5.3 계약·정산 리스크 흡수 조건 (04단계 딜 리뷰 반영)

- **가용성 크레딧 산정식 별도 정의(REQ-OPS-01)**: RFP 6-1 크레딧 표는 트리거가 "통지 SLA 준수율"  
  단일 지표뿐이어서 가용성 미달 시 크레딧 산정 경로가 부재함. SRS 인수 조건으로 가용성 구간별  
  크레딧(예: 99.95% 미만~99.9% 5% / 99.9% 미만~99.5% 10% / 99.5% 미만 15%)을 별도 정의하고,  
  통지 준수율 크레딧과의 중복 적용·합산 규칙을 계약 확정 시 합의함(연 누적 상한 15% 준용).  
- **AI 재학습 귀속 판정 절차(REQ-BIZ-05a·REQ-TEC-05)**: 백테스트 오차율 ±15% 초과 시 원인 판정  
  절차를 명문화함 — 판정 주체(발주사·벤더 공동 기술 검토), 입증책임(벤더가 재학습·데이터·로그 제시),  
  판정 기한(사고 종료 후 10영업일), 판정 결과: 구현결함 기인 → 무상 하자, 정상 운영 중 drift 기인  
  → MRC 유상 재학습.  
- **ETR 갱신 성실성 면책(REQ-BIZ-03)**: ETR 미갱신이 MNO 원천 복구정보 미제공에 기인함이 수신 로그로  
  입증되면 통지 준수율 크레딧에서 면책함(벤더 통제 불가 사유).  
- **정상 확인(이상 없음)**: 서비스 크레딧 연 누적 상한 15%, 지체상금 총액 cap NRC 30%, MNO 망 복구  
  MTTR 벤더 SLA 배제(관찰 지표) — 3항목은 통제가 적절히 설계됨(추가 조치 불요).

## 5.4 인수 판정 서술 예시 (Given-When-Then)

- **예1(통지 SLA)**: Given 월간 전체 통지 건이 자동 타임스탬프로 기록된 상태 / When 월 마감에 통지  
  SLA(수신~통지 1분·인지~담당자 5분) 준수 건수를 집계 / Then 준수율 ≥ 98%이면 합격, 미달 시 6-1  
  크레딧 표 적용. [REQ-OPS-02]  
- **예2(가용성)**: Given 이중화 구성·서비스 중단 시간 자동 수집 / When Availability=MTBF/(MTBF+MTTR)  
  월 집계 / Then ≥ 99.95%(월 다운타임 ≤ 21.6분)이면 합격, 미달 시 5.3 가용성 크레딧 산정식 적용.  
  [REQ-OPS-01]  
- **예3(AI 예측·확률형)**: Given 과거 사고 전수 백테스트 / When 각 사고 영향 규모를 예측 / Then 건별  
  오차율 ±15% 이내 건을 합격 집계·리포트하고 표본 부족 시 룰 폴백을 병행. [REQ-BIZ-05a]
