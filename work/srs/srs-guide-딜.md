## S6. 인수 기준(Acceptance Criteria)

> 담당: 딜(가격·계약 매니저). 다루는 SRS 항목: 측정 가능한 완료 기준 서술 규칙(기간·임계값·판정방법),  
> Given-When-Then 형식, SLA·서비스 크레딧·하자/유지 경계 등 정산 분쟁 소지 항목의 계약 연계.  
> 근거: 확정 RFP 2-3(운용 SLA)·6(계약조건)·부록 A(RTM). 원칙 "측정할 수 없으면 요구사항이 아님".  
> 상세 근거·출처는 정보수집 파일 `work/srs/srs정보수집-딜.md` 참조.

### (a) 작성 원칙

- 인수 기준은 FR·NFR이 "완료·합격"으로 판정되는 조건을 사전 합의로 고정하는 명세임. 구현 방식이  
  아닌 관찰 가능한 행위·결과에 초점을 두며, 판정 주체가 바뀌어도 동일하게 해석되어야 함.  
- 모든 인수 기준은 [지표 + 임계값 + 측정조건·판정방법] 3요소를 갖춤. 모호어("빠름·안정적·신속히")는  
  수량화하여 재작성함. 측정할 수 없으면 인수 기준이 아님.  
- 서술 형식은 Given-When-Then(선행조건→행위→관찰 결과)을 기본으로 하며, Then 절은 반드시 임계값·  
  시간·건수로 종결함. SMART(구체·측정가능·달성가능·관련성·기한)를 충족함.  
- 판정에 필요한 4요소를 명시함: ① 측정 지표 정의(산식) ② 임계값 ③ 측정 방법·데이터 원천  
  (자동 타임스탬프·로그) ④ 측정 주기·집계 단위(월·건별).  
- SLA·서비스 크레딧·하자/유지 경계 등 정산 분쟁 소지 항목은 인수 기준과 계약 조항을 1:1로 연계함.  
  벤더 통제 가능 지표만 크레딧·페널티 대상으로 하고, 관찰 지표(MNO 망 복구 MTTR 등)는 인수·페널티에서 배제함.  
- 인수 기준은 RTM을 통해 RFP 요구ID ↔ SRS ↔ 검증방법(PoC·부하시험) ↔ 계약 조항으로 추적 연결함.

### (b) 통신사 B2B 예시·문구

- 인수 기준 서술 예시 1 (통지 SLA 준수율 — REQ-OPS-02 근거, 서비스 크레딧 연계)  
  - Given: 월간 집계 대상 기간의 전체 통지 건이 자동 타임스탬프로 기록된 상태에서  
  - When: 월 마감 시점에 통지 SLA(수신~통지 1분·인지~담당자 5분) 준수 건수를 집계할 때  
  - Then: 준수율 = (준수 건 / 전체 통지 건) ≥ 98%이면 합격, 미달 시 서비스 크레딧을 적용함  
  - **인수 판정**: 월 통지 SLA 준수율 = 준수건/전체통지건 ≥ 98%, 자동 타임스탬프로 측정,  
    미달 시 RFP 6-1 크레딧 표(97~98% 5% … 93% 미만 20%) 적용 — REQ-OPS-02 근거  
- 인수 기준 서술 예시 2 (가용성 — REQ-OPS-01 근거, 서비스 크레딧 연계)  
  - Given: 통지 시스템이 이중화 구성으로 운영되고 서비스 중단 시간이 자동 수집되는 상태에서  
  - When: 월간 가용성을 Availability = MTBF/(MTBF+MTTR) 산식으로 집계할 때  
  - Then: 가용성 ≥ 99.95%/월(월 허용 다운타임 ≤ 약 21.6분)이면 합격, 미달 시 크레딧을 적용함  
  - **인수 판정**: 월 가용성 ≥ 99.95% & 월 다운타임 ≤ 21.6분, 월간 가용성 리포트로 측정 — REQ-OPS-01 근거  
- 인수 기준 서술 예시 3 (채널 자동 전환 — REQ-TEC-03 근거, 통지 준수율 반영)  
  - Given: API·이메일·SMS 다중 채널이 정상 운영 중인 상태에서  
  - When: 임의 채널을 강제 차단하는 전환 시험을 수행할 때  
  - Then: 대체 채널 자동 전환이 30초 이내 완료되면 합격 — REQ-TEC-03 근거, 미달 시 통지 준수율에 반영  
- SLA·서비스 크레딧 표 (RFP 6-1 준용, 벤더 통제 가능 지표 기준선)

| 월 통지 SLA 준수율 | 서비스 크레딧(월 MRC 대비) | 인수·정산 판정 |
|--------------------|-----------------------------|----------------|
| 98% 이상 | 없음 | 목표 달성(합격) |
| 97% 이상 ~ 98% 미만 | 5% | 크레딧 적용 |
| 95% 이상 ~ 97% 미만 | 10% | 크레딧 적용 |
| 93% 이상 ~ 95% 미만 | 15% | 크레딧 적용 |
| 93% 미만 | 20%(월 상한) | 최소목표 미달 |

- 크레딧은 월별 산정·적용, 연간 누적 상한 = 연 MRC의 15%. 징벌이 아닌 개선 유인으로 설계함  
  (기대목표 98% / 최소목표 93% 이원화).  
- 하자보수(무상 1년) vs 유지관리(유상) 인수 경계 문구 (RFP 6-3 준용)  
  - 무상 하자보수: 사업 완료일로부터 1년, "오류·결함 수정"에 한정. 인수 시 하자 목록·재현 절차를 첨부함.  
  - 유상 유지관리: 추가개발·기능개선·환경변경·추가 교육은 MRC/별도로 구분. 경계 판정 기준을 인수 기준에 명문화함.  
- AI 재학습 귀속 경계 인수 판정 (RFP 6-3·REQ-BIZ-05a 근거, 분쟁 예방)  
  - **인수 판정**: AI 영향범위 예측 백테스트 오차율이 과거 사고 전수 대비 ±15% 이내이면 합격  
  - **귀속 판정**: 오차율 ±15% 초과 시 원인이 구현결함이면 무상 하자, 정상 운영 중 drift 기인이면 MRC 유상 재학습  
- ETR 갱신 성실성 (REQ-BIZ-03 근거): ETR 15분 주기 갱신 준수율 ≥ 98%, 갱신 타임스탬프로 측정,  
  미달 시 통지 준수율에 반영. MNO 망 복구 MTTR은 관찰 지표로 인수·페널티에서 배제함.

### (c) 작성 체크리스트

- [ ] 모든 인수 기준이 [지표 + 임계값 + 측정조건·판정방법] 3요소를 갖췄는가  
- [ ] 모호어를 배제하고 Then 절을 임계값·시간·건수로 종결했는가(Given-When-Then·SMART)  
- [ ] 측정 지표에 산식(예: 준수율=준수건/전체건, 가용성=MTBF/(MTBF+MTTR))을 명시했는가  
- [ ] 측정 방법·데이터 원천(자동 타임스탬프·로그)과 측정 주기·집계 단위를 명시했는가  
- [ ] SLA 인수 기준이 서비스 크레딧 표(RFP 6-1)와 수치·구간이 일치하는가  
- [ ] 벤더 통제 가능 지표만 크레딧·페널티 대상으로 하고 관찰 지표(MNO MTTR)를 배제했는가  
- [ ] 하자보수(무상 1년)/유지관리(유상) 경계와 AI 재학습 귀속 판정 기준을 인수 기준에 명문화했는가  
- [ ] 각 인수 기준이 RTM으로 RFP 요구ID ↔ 검증방법(PoC·부하시험) ↔ 계약 조항에 추적 연결되는가
