## 데모 리뷰

### HIGH (검증 불가·정합 파괴 — 데모/정산에서 즉시 문제)

- [오류] 11장: 결과 상태값이 통지·조회·AI·응대·리포트·가용성 등 직교 차원을 단일 enum에 혼재하고,  
  "통지 지연(1분 초과) ⊂ 인지~담당자 초과(5분 초과)"가 부분집합 관계로 동시 성립 가능한데도 12장은  
  결과 상태를 "11장 중 하나"의 단일값으로 규정함 → 동시 성립 시 상태 배정 불가·상호배타성 위배 →  
  11장에 상태 간 적용 우선순위(9.2 준용) 또는 다차원 상태 모델을 명시하여 단일값 배정 규칙을 확정.  
  [REQ-OPS-02][REQ-TEC-01]
- [모호] 6.1: "긴급/높음/보통 3등급"을 가르는 규칙표(영향 규모·서비스 조합별 임계값)가 문서 어디에도  
  정의되지 않아, AC "규칙표 대사 표본 100건 100% 일치"의 대상 규칙표 자체가 부재 → 데모에서 등급  
  판정 정오를 검증 불가 → 6.1에 등급별 임계값(예: 영향 국사/셀 수·가입자 규모 구간)을 표로 명시.  
  [REQ-BIZ-01]
- [누락] 6.6/11장: 벤더 기인 ETR 갱신 지연(MNO가 원천 제공했으나 15분 주기 갱신 실패) 대응 상태값이  
  11장에 없음 — "면책"은 MNO 미제공만 커버 → 11장에 "ETR 갱신 지연(원천 수신 후 15분 주기 초과)"  
  상태를 추가하여 벤더 통제 구간 미준수를 측정·집계. [REQ-BIZ-03]
- [모호] 6.4/16장: "표본 부족 시 룰 폴백" 전환 임계값(표본 건수)이 미정의 → 룰 폴백 발동 시점 검증 불가  
  → RFP 재학습 근거(신규 사고 누적 10건, REQ-TEC-05)를 활용해 "학습 표본 N건 미만" 임계값을 6.4에  
  수치로 명시. [REQ-BIZ-05a][REQ-TEC-05]
- [누락] 6.5: 6.5를 "확률형"으로 표기했으나 품질·정확도 측정 임계값(오차율·룰 폴백)이 없고 적시성  
  (10분·추론 3초)만 존재 → 6.4 확률형 4요소(오차율·표본·룰폴백·HITL)와 불일치 → 6.5에 응대 초안  
  품질 AC(예: HITL 검수 채택률 또는 룰/템플릿 폴백 조건)를 추가하여 확률형 서술을 일관화.  
  [REQ-BIZ-05b][REQ-TEC-05]

### MED (모호·정합 공백 — 인수·리포트 단계 재협의 유발)

- [모호] 6.3/16장: "소규모 집계로 개인 특정 가능 구간" 상향집계 임계값(k-익명성 최소 집계 건수)이  
  미정의 → 상향집계 발동 시점 검증 불가 → 6.3 또는 별첨 마스킹 규칙에 최소 집계 건수(예: k) 임계값을  
  수치로 명시. [REQ-BIZ-02][REQ-TEC-04]
- [오류] 10.3/11장: 10.3 지연 상태값 4단계(대기/진행/완료/실패)와 11장 결과 상태값이 별도 모델로  
  존재하며 정합 관계 미정의 → 12장 "결과 상태(11장 중 하나)"에 10.3 상태 포함 여부 불명 → 11장 표에  
  지연 상태를 흡수하거나 두 모델의 매핑(지연 상태=결과 상태 하위)을 11장에 명문화. [REQ-OPS-02]
- [모호] 16장(반복 수신): "심각도 보통이더라도 임계 시간 내 반복 수신 시 상향 재판정"의 반복 횟수·  
  시간 임계값이 미정의 → 재판정 발동 조건 검증 불가 → 6.1/16장에 "N분 내 M회 이상 동일 사고" 수치를  
  명시. [REQ-BIZ-01]
- [누락] 6.7/11장: "사고 종료 판정"(3영업일 리포트 SLA 기산점)의 측정 가능 정의가 없음 → 리포트 지연  
  판정 시작 시점이 모호 → 6.6 복구 상태(완료)와 연계하여 "사고 종료=복구 상태 완료 확정 시각" 등  
  기산 기준을 6.7에 명시. [REQ-BIZ-04][REQ-BIZ-03]
- [누락] 16장: 가용성 미달(11장 상태 존재) 및 AI 재학습 귀속(drift vs 구현결함, ±15%)에 대한 X-RAY  
  예외 케이스가 부재 — A4 딜 최대 분쟁 지점 → 16장에 "가용성 미달 시 크레딧 산정 경로", "±15% 초과가  
  drift 기인이면 재학습·구현결함이면 하자보수" 예외 2건을 추가(별도 인수기준장 신설 금지, 16장 흡수).  
  [REQ-OPS-01][REQ-TEC-05]

### LOW (개선 — 완결성 보강)

- [누락] 6.4/6.5: RFP 재학습 트리거(신규 사고 누적 10건 재학습 + 분기 성능점검, REQ-TEC-05)가 6장  
  AI 판정·5.3 전처리에 미반영 → 6.4/6.5 또는 5.3에 재학습 주기·성능점검 기준을 관찰·측정 항목으로  
  명시. [REQ-TEC-05]
- [개선] 6.6: "15분 주기 연속 갱신"의 준수 허용오차(예: 주기당 ±N분)가 미정의 → 6.6에 주기 준수  
  판정 허용오차를 명시하여 벤더 기인 지연(상기 HIGH 신규 상태)과 정상 갱신의 경계를 측정 가능화.  
  [REQ-BIZ-03]
