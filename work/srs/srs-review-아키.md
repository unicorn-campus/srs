## 1. 총평

- **전반**: NFR 12건이 명세표(SEC4.1.1)에 [지표+목표값+측정조건] 3요소 골격으로 정형화됐고, RFP Capacity표·  
  NFR·장해대책 수치와 **전건 일치**함(대사 결과 §2.2). 04단계 아키 리뷰에서 누락 지적됐던 **DR(RTO 1시간/  
  RPO 15분)이 NFR-A04로 명시**되어 해소됨. REQ-TEC-01 능동 헬스체크 전제, REQ-TEC-02 CQRS·사전집계 설계  
  방향은 **정석적이며 타당**함.  
- **핵심 리스크**: 다만 "실시간 변동 데이터 + 캐시/사전집계 + 100% 대사"(NFR-P03), "영향범위 산출 입력  
  데이터 출처"(데이터 요건), "가용성 21.6분/월 ↔ DR 복구 1시간"(NFR-A01↔A04)의 **세 정합성 이슈**는 지금  
  명세로는 검수 시점에 정산 분쟁 또는 설계 확정 불가로 귀결될 소지가 있어 **수정 필수**로 판정함.  
- **판정**: 측정 가능성·수치 정합성은 양호하나, **아키텍처 정합성(성능↔정확성, 가용성↔DR)과 데이터 소싱  
  실현가능성**에서 보강이 필요함. 수정 필수 3건·권장 6건.

## 2. NFR·데이터 실현가능성·측정 가능성 점검

### 2.1 NFR 12건 3요소·측정 가능성 점검표

> 판정: ○=3요소 충족·측정 가능/타당, △=3요소는 있으나 판정 기준·산식·정합성 보강 필요.

| NFR-ID | 지표 | 목표값 | 측정조건 | 3요소 | 판정 | 비고(지적ID) |
|--------|------|--------|----------|:---:|:---:|------|
| NFR-P01 | 수신~통지 지연 | 1분 이내 | 수신~통지 종단, 월, 정상·피크 | 有 | △ | 측정구간 표현·최악예산(A-R02·A-R01) |

- 요약: 12건 중 **7건 ○(타당)**, **5건 △**(P01·P02·P03·A01·A03). △ 중 P03·A01은 아키텍처 정합성  
  이슈로 **수정 필수**, 나머지(P01·P02·A03)는 판정 기준·표현 보강으로 **권장**.

### 2.2 RFP 수치 대사 결과

- **전건 일치**: 통지 1분·성공률 99.9%(REQ-TEC-01), 조회 평균2초&P99 3초·300 TPS(REQ-TEC-02), 채널 전환  
  30초(REQ-TEC-03), OAuth2.0·TLS1.2+(REQ-TEC-04), 담당자 500계정·이력 3년·무중단 Scale-out(2-2 Capacity),  
  가용성 99.95%/월 21.6분(REQ-OPS-01), DR RTO 1h/RPO 15m(2-2 Capacity) — **12개 지표 전부 RFP와 일치**하며  
  임의 수치 없음.  
- **DR 반영 확인(중요)**: RFP 2-2 Capacity표·장해대책에만 있던 DR 수치가 SRS에서 NFR-A04로 승격·명세됨.  
  04단계 아키 리뷰의 "DR 판정 누락" 지적이 실제 해소됨을 확인함(SEC4.1.4).  
- **귀속 확인**: Capacity표 항목(계정·이력·Scale-out·DR)을 신규 ID 없이 REQ-TEC-06으로 부기한 처리는 원칙  
  (신규 ID 미생성)에 부합하여 **타당**함. 단 DR의 SLA 연결 경로는 §3 A-M03에서 별도 지적함.

### 2.3 REQ-TEC-01 페일오버 전제(능동 헬스체크) 검토 — **대체로 타당**

- **타당한 부분**: 능동(active) 헬스체크로 채널 상태를 사전 감지·배제하고 병렬/우선순위 전송하는 설계는,  
  요청 실패 후 타임아웃을 기다리는 수동 페일오버 대비 전환 지연이 작아 **30초 전환 목표에 부합**함.  
  "수동 타임아웃형 배제" 명시(SEC4.1.2)는 04단계 기술 리스크를 정확히 반영한 판단으로 **타당**함.  
- **보강 필요**: RFP REQ-TEC-01은 "채널 전환 30초가 1분 예산의 절반을 소진함에 유의"라 명시(RFP 2-2).  
  SRS는 "30초가 1분을 침범하지 않게 설계"로만 서술해, **런타임 전환이 실제 발생하는 최악 케이스의 예산  
  배분(전환 30초 + 잔여 30초 전송)과 헬스체크 감지 지연 상한**이 없음. 이 경우 1분 SLA 판정이 모호함  
  → A-R01(권장).

### 2.4 REQ-TEC-02 조회 성능 설계(CQRS·캐시 신선도) 검토 — **방향 타당, 정합 기준 필수**

- **타당한 부분**: CQRS 읽기 최적화 + 사전집계 + 캐시로 피크 300 TPS에서 평균2초&P99 3초를 확보하는  
  접근은 대규모 조회 폭주 대응의 정석으로 **타당**함. 버스트 헤드룸·피크 부하시험 확증도 적절함.  
- **핵심 리스크(필수)**: 영향범위는 사고 진행 중 실시간 변동하는 데이터임. 캐시/사전집계와 FR-조회-01의  
  "표본 100건 100% 대사"는 **본질적으로 긴장 관계**임. "양립하도록 관리"라는 서술만으로는 검수 시 판정  
  불가 — **캐시 최대 지연(신선도 상한)·사전집계 갱신 주기·대사 기준 스냅샷 시각**의 정량 명세가 없으면  
  성능(캐시)과 정확성(100% 대사)이 실제로 양립 불가하거나 정산 분쟁 소지 → A-M01(필수).

### 2.5 데이터 요건(개인정보·무결성·연동) 검토

- **개인정보 집계 단위(타당, 기준 1건 보강)**: 개인 식별자 배제·집계 범위 제공(SEC4.2.2)은 실현 가능하며  
  타당함. 다만 소규모 집계("○○구 △△명 영향")의 **재식별 방지 최소 집계 크기(k-익명성 임계)**가 없어,  
  집계 수가 작을 때 안전성 확보조치 취지가 훼손될 수 있음 → A-R05(권장).  
- **영향범위 데이터 소싱(필수)**: 영향범위 = MNO 장애 지역/서비스 × 자사 가입자 분포로 산출되나, **입력  
  데이터 출처(자사 가입자DB vs MNO 제공 장애 정보)·조인 방식·수신 필드**가 데이터 요건에 없음. NFR-P03  
  (P99 3초)과 CQRS 사전집계 설계의 실현가능성이 이 소싱에 직접 의존하므로 확정 필요 → A-M02(필수).  
- **연동 규격(대체로 타당, 수신 이중화 1건 보강)**: Webhook 재전송·중복 제거·서명 검증, API 스키마·오류  
  코드, NTP 시각 동기(SEC4.2.3)는 **타당·우수**함. 단 통지 채널은 이중화(NFR-A02)했으나 **MNO 이벤트  
  수신 채널(Webhook) 장애·유실 감지 및 API 폴링 백업**이 없어 수신 단일점이 잔존함 → A-R06(권장).  
- **통지이력 무결성(타당)**: 해시체인·전자서명·WORM + 해시 대사(SEC4.2.4)로 NFR-S02·C03과 연계한 설계는  
  **타당**함. 3년 전건 검증은 해시체인 특성상 체인 검증으로 담보 가능(방법 명시는 선택).

## 3. 지적사항 목록

> 표기: [대상] 문제 → 조치. 필수=아키텍처 정합성·실현가능성상 미보강 시 설계 확정 또는 검수 판정 불가.

### 3.1 수정 필수 (3건)

- **A-M01 [NFR-P03·FR-조회-01] 캐시 신선도 ↔ "100% 대사" 정합 기준 부재**  
  실시간 변동 데이터에 캐시·사전집계를 쓰면서 표본 100건 100% 일치를 요구하나 정합 기준이 없음.  
  → **조치**: ① 캐시 최대 지연(신선도 상한, 예 N초) ② 사전집계 갱신 주기 ③ 대사 기준 스냅샷 시각(동일  
  기준시점 대사)을 SEC4.1.2·SEC5(AC)에 정량 명세. "100% 대사"를 "동일 스냅샷 기준 100% 일치"로 재정의.

- **A-M02 [데이터 요건·NFR-P03] 영향범위 데이터 소싱·조인 규격 누락**  
  영향범위 산출의 입력 데이터 출처·조인 방식·수신 필드가 미정의. P99 3초·사전집계 설계의 전제가 불명.  
  → **조치**: SEC4.2.1/4.2.3에 영향범위 입력원(자사 가입자 지역·서비스 매핑 DB × MNO 장애 이벤트의 영향  
  지역/서비스 필드)과 조인·집계 규격, MNO 제공 여부(가정/제약)를 명세. MNO 미제공 시 대체 산출식 정의.

- **A-M03 [NFR-A01 ↔ NFR-A04] 가용성 예산(21.6분/월)과 DR 복구(RTO 1시간)의 정합·취급 미명세**  
  DR 1회 발동 시 최대 60분 다운으로 그 달 가용성 예산(21.6분)을 자동 초과 → 크레딧 산정 모순 발생.  
  → **조치**: NFR-A01 측정조건에 **재해(DR 발동) 구간의 가용성 SLA 취급(불가항력 면책·별도 집계 여부)**을  
  명시하고, DR 미달 시 SLA 연결 경로(현재 REQ-TEC-06 귀속으로 가용성·크레딧과 단절)를 SEC5·SEC6에 연결.

### 3.2 권장 (6건)

- **A-R01 [NFR-P01] 통지 1분 최악 케이스 예산 배분 명세**  
  런타임 채널 전환 발생 시 "전환 30초 + 전송 30초" 1분 예산 배분과 헬스체크 감지 지연 상한을 명세 →  
  전환 발생 시나리오에서 1분 SLA 판정 명확화(RFP "30초=1분의 절반 소진" 유의사항 반영).

- **A-R02 [NFR-P01] 측정조건 "수신~통지 종단" 표현 정정**  
  1분은 시스템 전송 구간(SEC1.4 정의)인데 명세표 측정조건은 "종단"으로 기재 → 5분(인지~담당자 종단)과  
  혼동 소지. "수신~통지(시스템 전송 구간)"로 표기 통일(SLA 정산 분쟁 예방).

- **A-R03 [NFR-P02] 전송 성공률 산식 정의 부재**  
  다중 채널(API+SMS+이메일)에서 "성공" 판정(1개 채널 성공 vs 전 채널)·분모/분자(통지 건 기준 vs 채널  
  기준)가 없음 → 99.9% 측정 산식을 SEC4.1.2·SEC5에 명시.

- **A-R04 [NFR-A03] "무중단" 정량 판정 기준 부재**  
  목표값이 "지원(무중단)"으로 정성적 → 노드 증설 중 조회 성공률·P99 유지(예: 증설 중 성공률 99.9% 이상,  
  P99 3초 유지)로 무중단을 정량 판정하도록 AC 보강.

- **A-R05 [데이터 요건·개인정보] 집계 최소 크기(k-익명성) 기준 부재**  
  소규모 집계의 재식별 방지 최소 집계 크기(예 N명 미만 구간화/마스킹)를 SEC4.2.2에 명세 → 안전성 확보조치  
  실효성 담보.

- **A-R06 [연동 규격] 수신 채널(Webhook) 이중화·유실 감지 누락**  
  통지 채널은 이중화됐으나 MNO 이벤트 수신 채널 장애·유실 대비 API 폴링 백업·유실 감지(heartbeat)가 없음  
  → SEC4.2.3에 수신 경로 이중화 명세(통지 1분 SLA의 시작점인 수신 신뢰성 확보).

## 4. 결론

- **판정**: NFR 측정 가능성(3요소)·RFP 수치 정합성·페일오버/조회 성능 설계 방향은 **양호·타당**하며, 특히  
  DR(NFR-A04) 반영으로 04단계 누락 지적이 해소됨. 반면 **아키텍처 정합성(성능↔정확성, 가용성↔DR)과  
  영향범위 데이터 소싱 실현가능성**에서 보강이 필요함.  
- **수정 필수 3건**: A-M01(캐시 신선도↔100% 대사 정합), A-M02(영향범위 데이터 소싱·조인 규격),
